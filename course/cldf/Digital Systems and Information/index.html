    1.1 Digital Systems¶

    Takes a set of discrete information inputs and discrete internal information(system state) and generates a set of discrete information outputs.

    采用一组离散信息输入和离散信息内部信息（系统状态）并生成一组离散信息输出。

    1.1.1 Types of Digital Systems¶

    • no state present: Combinational Logic System.Output = Function(Input)
    • state present: Sequential System
    • state updated at discrete time
      • Synchronous Sequential System同步顺序系统
    • state updated at any time
      • Asynchronous Sequential System异步顺序系统
    • state = Function(Input,State)
    • Output = Function(State) or FUnction(State,Input)

    注意点：State 既与 input 有关，还与State（先前）有关

    每次的 state 只是作为了 input 到 output 的中间量

    1.1.2 Embedded Systems 嵌入式系统¶

    analog input：

    • A-to-D：把模拟信号转化为数字信号（处理器不能处理模拟信号）
    • D-to-A：把处理后的数字信号转化为模拟信号

    example: 对空调而言，input是温度传感器，输入的是一个电压值，空调如何理解电压的变化

    1.2 Information Representation¶

    Binary values are represented abstractly by:

    • digits 0 and 1
    • words(symbols) False(F) and True(T)
    • words(symbols) Low(L) and High(H)
    • and words On and Off

    • 模拟信号：在值和时间上连续
    • 异步：在时间上连续，在值上离散
    • 同步：在值和时间上连续

    1.3 Number Systems - Representation¶

    \[ \begin{aligned} & A \ number \ with \ radix(基数) \ is \ represented \ by \ a \ string \ of \ digits: \\ & A_{n-1}A_{n-2}\dots A_{1}A_{0}.A_{-1}A_{-2}\dots A_{-m}\\ & Number = (\sum_{i = 0}^{i = n-1}A_i \times r^i)+(\sum_{j = -m}^{j= -1}A_j \times r^j)\\ \end{aligned} \]

    1.5.1 Binary Codes for Decimal Digits 十进制数字的二进制代码¶

    • 8421: Binary Code Decimal(BCD 码) 每个码都有位权
    • Excess3(余3码): 8421 + 3 得到 Excess3 的值. 编码成对出现, i 和 9 - i 互反. 在通信时很有用处.
    • 84-2-1: 有权编码, 类似 8421 码, 也是成对出现.
    • Gray 格雷码: 相邻两个码之间只有一位不同.

    1.5.2 Binary Coded Decimal（BCD码）¶

    • The BCD code is the 8,4,2,1 code
    • This code is the simplest, most intuitive binary code for decimal digits and uses the same powers of 2 as a binary number, but only encodes the first ten values from 0 to 9.
    • 此代码是最简单、最直观的十进制数字二进制代码，使用与二进制数相同的 2 的幂，但仅编码从 0 到 9 的前十个值。

    不要将十进制转换为二进制数（conversion of a decimal number to a binary number）与用二进制代码对十进制数进行编码（coding a decimal number with a Binary code）相混淆

    • \(13_{10} = 1101_{2}\) `This is conversion
    • \(13 \Leftrightarrow 0001 | 0011\) This is coding

    $$ (185) {10} =(0001 1000 0101)

    =(10111001) _2 $$

    • BCD码可以表示0到9数字，一旦大于9，就需要分别表示两位数字
    • 可以计算，但要修正。（超过 9 要加 6）

    1.5.3 ASCII Character Codes¶

    常用的 ASCII Code:

    • 0 <-> \(30_{16}\)
    • A <-> \(41_{16}\)
    • a <-> \(61_{16}\)
    • Delete (DEL) is all bits set.

    1.5.4 Parity Bit Error-Detection Codes¶

    在信号传输过程中，可能由于环境干扰等原因，出现各种信号抖动，所以为了保证数据的可信度，我们需要一个错误检测机制。

    • Redundancy (e.g. extra information), in the form of extra bits, can be incorporated into binary code words to detect and correct errors.

    冗余（例如额外信息）以额外位的形式可以合并到二进制码字中以检测和纠正错误。

    • A simple form of redundancy is parity, an extra bit appended onto the code word to make the number of 1’s odd or even. Parity can detect all single-bit errors and some multiple-bit errors.

    冗余的一种简单形式是奇偶校验，即在码字上附加一个额外位，以决定 1 的数量为奇数或偶数。奇偶校验可以检测所有单位错误和一些多位错误。

    • A code word has even parity if the number of 1’s in the code word is even.

    如果码字中 1 的个数为偶数，则该码字具有偶奇偶校验。

    • A code word has odd parity if the number of 1’s in the code word is odd.

    如果码字中 1 的数量是奇数，则该码字具有奇奇偶校验。

    The codeword "1111" has even parity and the codeword "1110" has odd parity.

    1.6 Gray Code格雷码¶

    Gray ： 相邻两位变化的仅有1bit

    • 编码器盘包含不透明区域和透明区域。
    • Opaque表示0 ,clear表示1。光通过编码器的每一个环，与编码的一个位相对应，照射在传感器上，产生一个0或1

    • 旋转编码器, 测量转轴转向的角度.
    • 三个传感器不会在同一条直线(安装误差), 存在中间状况.
    • 但格雷编码盘, 相邻编码只差一位, 对结果不会带来影响.

    如图，有三个传感器。对于二进制编码，中间会出现100->110->111的中间状态。而对于Gray编码，任何时候都只能产生正确的信号。

    因为，Gray编码相邻变化的只有1bit，说明中间的变化过程已经确定，仅有1bit发生改变。
