    Registers, Microoperations and Implementations

    6.1 Register¶

    Register – a collection of binary storage elements

    寄存器——二进制存储元素的集合

    上图所示为2位寄存器

    一共有 \(2^2 = 4\)中状态

    输出函数\(Y1 = A1(t), Y0 = A0(t)\)，输入组合四种

    输出方程 \(Y1 = A1， Y0 = A0\)，输出结果与输入In0，In1无关，输入改变的仅是下一时刻状态，继而影响输出结果属于Moore模型

    如果按照这种方法，设计寄存器，会发现状态数和输入组合数呈现指数型增长，随着n变大，不堪重负

    解决方法：

    1. Add predefined combinational circuits to registers(e.g. To count up, connect the register flip-flops to an incrementer) 将预定义的组合电路添加到寄存器（例如，要向上计数，请将寄存器触发器连接到增量器）
    2. Design individual cells using the state diagram/state table model and combine them into a register 使用状态图/状态表模型设计单个单元并将它们组合到寄存器中 把寄存器拆成单位的来设计，再考虑多位的拼接，最后组成多位寄存器。

    6.1.1 Register Storage and Load Enable¶

    Expectations: 期望：

    • A register can store information for multiple clock cycles 一个寄存器可以存储多个时钟周期的信息
    • To “store” or “load” information should be controlled by a signal “存储”或“加载”信息应由信号控制

    但在刚刚的例子中，每个周期都会保存新的数据，不能做到在多个周期保留数据，因此是不行的。

    Realizing expectations: 实现期望：

    • Use a signal to block the clock to the register 使用信号阻止寄存器的时钟
    • Use a signal to control feedback of the output of the register back to its inputs 使用信号控制寄存器输出到其输入的反馈
    • Use other SR or JK flip-flops, that for (0,0) applied, store their state 使用其他 SR 或 JK 触发器，对于应用的 (0,0)，存储它们的状态

    Load is a frequent name for the signal that controls register storage and loading

    Load 是控制寄存器存储和加载的信号的常用名称

    • Load = 1: Load the values on the data inputs Load = 1 ：加载数据输入上的值（加载外部新数据）
    • Load = 0: Store the values in the register Load = 0 ：将值存储在寄存器中（保留原有的数据）
    1. 使用信号阻断时钟，此时时钟信号不产生上升沿，D触发器也就不会改变
    2. 添加保持功能，从D触发器的输出结果引出一条线连接到输入选择部分
    3. 添加控制保持还是加载的信号Load

    Solution 解决方法

    • Registers with Clock Gating 带时钟门控的寄存器,相当于block clock

    The \(\overline {Load}\)(bar 表示低电平有效) signal enables the clock signal to pass through if 0 and prevents the clock signal from passing through if 1. (与 Load 相反)

    \(\overline {Load}\)（bar表示低电平有效）信号如果为0则允许时钟信号通过，如果为1则阻止时钟信号通过。（与 Load 相反）

    For Positive Edge-Triggered or Negative Pulse Master-Slave Flip-flop: 对于正边沿触发或负脉冲主从触发器：

    \(Gated \ Clock = Clock + \overline{Load}, 当\overline{Load}信号为1时，时钟信号不随外部时钟改变，这时保持原有的数据\)

    ​ Clock Skew 始终偏差

    ​ 问题在于我们是同步时序电路实现，要求时钟统一提供，这样所有的触发器会在同一时间 完成操作。使用门控时钟的方式会带来更多的触发时间，时序电路从同步变为了异步的时 序电路（触发有前有后）

    • Registers with Load-Controlled Feedback 具有负载控制反馈的寄存器

    添加load信号，控制选择的输入是In，还是输出结果Y。分别对应加载和保持

    Run the clock continuously, and Selectively use a load control to change the register contents.

    连续运行时钟，并有选择地使用负载控制来更改寄存器内容。

    1. 引入一个2to1Mux，Load作为选择信号
    2. 当Load = 0时，加载寄存器当前的值
    3. 当Load = 1时，加载输入值，Y1对应In1，Y0对应I0

    6.1.2 Register Transfer Operations¶

    • Register Transfer Operations – The movement and processing of data stored in registers 寄存器传输操作 – 存储在寄存器中的数据的移动和处理
    • Three basic components: 三个基本组成部分：
    • set of registers(源寄存器，目标寄存器)
    • operations 运营
    • control of operations 操作控制
    • Elementary Operations -- load, count, shift, add, bitwise "OR", etc. 基本运算——加载、计数、移位、加法、按位“或”等。

    Register Notation 寄存器符号

    • Letters and numbers – denotes a register （R2，PC，IR） 字母和数字 – 表示寄存器
    • Parentheses ( ) – denotes a range of register bits \((R1(1),PC(7 : 0),PC(L))\) 括号 ( ) – 表示寄存器位的范围
    • Arrow (←) – denotes data transfer (*e.g.* \(R1←R2,PC(L)←R0)\) H 代表高位, L 代表低位(如 PC(L),PC(H) 分别代表 PC 的高八位和低八位)
    • Comma – separates parallel operations 逗号 – 分隔并行操作
    • Brackets [ ] – Specifies a memory address \((e.g. R0←M[AR],R3←M[PC])\) 寻址 括号 [ ] – 指定内存地址

    6.1.2.1 Conditional Transfer 有条件传输¶

    \(If （K1 = 1）then \ R2 \leftarrow R1 \ is \ shortened \ to \ K1:(R2 \leftarrow K2)\)

    如果 K1=1 那么将 R1 的信号传给 R2, 我们可以写为 K1:(R2←R1), 其中 K1 是一个控制变量，表明条件执行的发生是否发生。

    R2等待上升沿、下降沿到来，才会接受新的数据

    Control Expression 控制表达式

    • 控制表达式出现在左边，用冒号分隔
    • 逻辑为1表示运算发生，逻辑为0表示运算不发生

    6.1.2.2 Micro operation微操作¶

    • 微操作出现在冒号的右侧，包括传输，算术，逻辑，移位

    这边特别注意一下: 左边的是逻辑运算，+表示或，右边的微操作，逻辑操作或会用V表示

    算术操作

    逻辑操作

    移位操作

    不足位数采用0填充

    6.1.3 Register Transfer Structures¶

    • Multiplexer-Based Transfers 由多路选择器选择输入
    • Bus-Based Transfers 总线传输输入和输出结果
    • Three-State Bus 三态门使能与否控制是否作为输入
    • Other Transfer Structures

    6.1.3.1 Multiplexer-Based Transfer 基于多路复用器的传输¶

    Multiple inputs are selected by a multiplexer dedicated to the register

    基于多路复用器的传输 - 由专用于寄存器的多路复用器选择多个输入

    控制加载的信号，\(K1 + K2 \overline K1 = K1 + K2\),能够简化电路

    n位的2选1多路复用器，输入到R0.当K1等于1时，选择输入到多路复用器1的R1.

    6.1.3.2 Multiplexer and Bus-Based Transfers for Multiple Registers¶

    Multiple inputs are selected by a shared multiplexer driving a bus that feeds inputs to multiple registers

    基于总线的传输 - 由驱动总线的共享多路复用器选择多个输入，该总线将输入馈送到多个寄存器

    此处，一共有三条Bus总线，互不干扰

    如何实现R0和R1的互换？

    • 先令S0和S1分别为0，此时R0处的总线通过S1的MUX，R1处的总线通过S0的MUX，

    • L0和L1同时变为1的时候，R0和R1的值会发生互换

    还可以实现R0的值传输到R1，R1的值传输到R2？

    • 令S1 = 0， S2 = 0
    • 再令L1 = L2 = 1

    A single bus driven by a multiplexer lowers cost, but limits the available transfers

    由一个单总线驱动，多路复用器降低了成本，但限制了可用的传输

    不能同时 R0←R1 R1←R0(类比：交换函数需要一个第三方变量)

    如何实现寄存器之间的传输？

    • 确定谁是被传输的数据，利用S1，S0所在的Mux进行选择后传输到总线
    • 在利用Load指令，加载到目标寄存器

    6.1.3.3 Three State Bus¶

    Multiple inputs are selected by 3-state drivers with outputs connected to a bus that feeds multiple registers

    三态总线 - 由三态驱动器选择多个输入，输出连接到为多个寄存器供电的总线

    三态门存在三个状态，0，1，高阻态，其中0，1需要使能（E = 1）

    寄存器的输出结果通过三态门连接到总线，控制目前是哪个寄存器的值在总线上

    如何实现R0传输到R1

    • E0 = 1， 将寄存器R0的值传输到总线
    • L1 = 1， Load信号启动，加载输出的数据

    同样无法实现寄存器数据的同时交换

    6.1.3.4 Other Transfer Structure¶

    Use multiple multiplexers, multiple buses, and combinations of all the above 其他传输结构 - 使用多个多路复用器、多个总线以及上述所有结构的组合

    6.1.4 Register Cell Design¶

    1. Register Cell Design

    Assume that a register consists of identical cells.(每个位执行相同操作)

    假设寄存器由相同的单元组成。（每个执行位操作相同）

    • Design representative cell for the register 设计寄存器的代表单元
    • Connect copies of the cell together to form the register 将单元格的副本连接在一起以形成寄存器
    • Applying appropriate "boundary conditions" to cells that need to be different and contract if appropriate 对需要不同并在适当时收缩的单元应用适当的“边界条件”
    1. Register Cell specification 寄存器单元规格

    控制寄存器的输入组合。对于无编码器encoder，执行的操作无非是Load，shift，Add种其一。对于有编码器enable，注意enable输出组合作为控制Mux的输入。

    6.2 Shift Register¶

    在最简单的情况下，移位寄存器只是一组按行连接的 D 触发器

    Data input, In, is called a serial input or the shift right input.

    Data output, Out, is often called the serial output(串行输出).

    此处使用D触发器，能够保证执行一轮移位后停止（上升沿停止）

    T0 is the register state just before the first clock pulse occurs

    T1 is after the first pulse and before the second

    6.2.1 Parallel Load Shift Registers 并行加载移位寄存器****¶

    By adding a mux between each shift register stage, data can be shifted or loaded

    通过在每个移位寄存器级之间添加多路复用器，可以移位或加载数据

    If SHIFT is low,\(D_A\) and \(D_B\) are replaced by the data on DA and DB lines, else data shifts right on each clock.

    如果 SHIFT 为低电平，(\(D_A\)\) 和 \(D_B\)将被 DA 和 DB 线上的数据替换（并行加载），否则数据在每个时钟上右移。

    缺点: 没有数据保持操作，需要多添加一个选择项(3-1 选择)

    当shifit为1的时候，进行移位操作

    当shift = 0， load = 0，进行数据保持操作

    当shift = 0， Load = 1时，进行加载操作，Q = D

    相当于此处在\(Q_i\)前面加一个多路选择器，选择是\(Q_{i-1},Q_{i+1},Q_{i},D_i\),分别对应右移，左移，保持，加载
