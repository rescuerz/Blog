    Binary Substraction

    3.4 Binary substraction 二进制减法¶

    3.4.1 Unsigned Subtraction¶

    • Subtract the subtrahend(减数) N from the minuend(被减数) M
    • If no end borrow occurs, then M >= N, and the result is a non-negative number and correct.
    • If an end borrow occurs, the \(N >= M\) and the difference $M - N + 2 ^n $ is subtracted from \(2 ^ n\), and a minus sign is appended to the result.

    3.4.1.1 complement¶

    • Diminished Radix Complement of N 反码(按位取反)

    • defined as \(r^n - 1 - N(r^n - 1)\)是 bits[n-1:0] 全为 1 的二进制数，用它减去 N 即可得到 N 按位取反的结果，即反码)

      因为反码加上本身得到的是全是1的二进制数

    • Radix Complement 2‘s complement 补码（按位取反再加一）

    • 也可以这样求补码：从右往左第一个 1 之前不变，此后其他位全部求反

    • defined as \(r^n - N\)

    • 补码 = 反码 + 1

    Subtraction is done by adding the complement of the subtrahend.减法通过加上减数的补码实现

    • Add the 2’s complement of the subtrahend N to the minuend M: \(M + (2^n - N)\)
    • if \(M > N\), the sum produces end carry \(2^n\),which is discarded. \(M - N \ remains\)
    • if \(M < N\), the sum does not produce an end carry. The result is \(2^n - (N - M)\),which is the 2’s complement of \(N - M\)

    • Signed Integer Representations: 第 n-1 位表示正负，后面 bits[n-2:0] 表示绝对值大小
    • Signed-Complement
    • Signed 1's Complement
    • Signed 2's Complement

    3.4.2.1 Signed-Magnitude Arithmetic 有符号数值算术¶

    • 检查三个符号位的奇偶性（两个操作数的符号位和加减法的符号位，我们一般认为加法是 0, 减法是 1）用于判断溢出 可能溢出的情况：正加正(000), 正减负(011), 负减正(101), 负加负(110).

    前两位是正负判断，最后一位是加减判断

    • If the parity of the three signs is 0:(overflow may happen)

    • Add the magnitudes.

    • Check for overflow (a carry out of the MSB)
    • The sign of the result is the same as the sign of the first operand.

    如果三个符号的奇偶性为0（也就是加法）：（可能发生溢出）将大小相加。检查是否溢出（MSB 进位） 结果的符号与第一个操作数的符号相同。

    • If the parity of the three signs is 1:

    • Subtract the second magnitude from the first.

    • If a borrow occurs: 如果发生借用： take the two’s complement of result and make the result sign the complement of the sign of the first operand. 取结果的补码，并使结果的符号为第一个操作数的符号的补码。
    • Overflow will never occur.

    1. 有符号整数，第一位符号位，确定正负，后面就是简单的计算

    2. 1的补码，如果是大于0的数，那么与有符号整数相同

    如果是小于0的数，就把相反数逐位取反

    1. 2的补码，计算方法你懂的，不多做解释

    3.4.2.2 Signed-Complement Arithmetic有符号补码算术¶

    • Addition:

    • Add the numbers including the sign bits, discarding a carry out of the sign bits (2's Complement), or using an end-around carry (1's Complement).

      将包含符号位的数字相加，丢弃符号位的进位（2 的补码）

    • If the sign bits were the same for both numbers and the sign of the result is different, an overflow has occurred.

      如果两个数字的符号位相同并且结果的符号不同，则发生溢出。

    • The sign of the result is computed in step 1.

    • Subtraction: Form the complement of the number you are subtracting and follow the rules for addition.

    2’s Complement Adder/Subtractor

    利用异或门，当 S=0 时异或门相当于保持另一个信号，当 S=1 时异或门相当于对另一个信号取反。

    对于异或门，A XOR 0 = 本身A，A XOR 1 = \bar A(A取反)

    当\(S = 1 \(时，此时执行减法，\)A-B = A + \bar B + 1\)，此时\(C_0 = 1\)

    当\(S = 0\)时，此时执行加法，\(A + B = A + B\)

    Overflow Detection

    溢出可能发生在：两个符号相同的操作数相加或者不同符号操作数的减法

    Simplest way to implement overflow \(V = C_n \oplus C_{n-1}, C_n\) 是溢出去的位，\(C_{n-1}\) 是运算后的符号位。
