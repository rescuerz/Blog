    Chapter 4 | The von Neumann Modle冯诺依曼模型¶

    4.1 Basic Components¶

    To get a task done by a computer, we need two things:

    • (a) a computer program that specifies what the computer must do to perform the task,

    • (b) the computer that is to carry out the task.

    The von Neumann model consists of five parts: memory, a processing unit, input, output, and a control unit.

    4.1.1 Memory¶

    • 当今大多数电脑都是16GB内存
    • consist of \(2^{34}\)​distinct memory locations,each of which is capable of storing eight bits of information.
    • 地址空间（address space）为 \(2^{34}\),寻址能力（addressability）是8bits
    • 16 = \(2^4\),1GB = \(2^{10} MB = 2^{20}KB=2^{30}B\),16GB = \(2^{34}B\)

    对于LC-3，We will see that the memory address space of the LC-3 is $2^16$, and the addressability is 16 bits.

    we first place the address of that location in the memory’s address register (MAR)地址寄存器 and then interrogate(询问) the computer’s memory. The information stored in the location having that address will be placed in the memory’s data register (MDR)数据寄存器. To write (or store) a value in a memory location, we first write the address of the memory location in the MAR, and the value to be stored in the MDR. We then interrogate the computer’s memory with the write enable signal asserted. The information contained in the MDR will be written into the memory location whose address is in the MAR

    我们首先将该位置的地址放入内存的地址寄存器（MAR）中，然后询问计算机的内存。存储在具有该地址的位置的信息将被放置在存储器的数据寄存器（MDR）中。要将值写入（或存储）到内存位置，我们首先将内存位置的地址写入 MAR 中，并将要存储的值写入 MDR 中。然后，我们通过置位写入使能信号来询问计算机的内存。 MDR中包含的信息将被写入MAR中地址的内存位置

    The value stored in that location can be changed, but the location’s memory address remains unchanged.

    4.1.2 Processing Unit¶

    • The processing unit in a modern computer can consist of many sophisticated（复杂的） complex functional units（复杂的功能单元）, each performing one particular operation(divide, square root, etc.).

    • ALUis the simplest processing unit

    • ALU is the abbreviation(缩写) for Arithmetic and Logic Unit, so called because it is usually capable of performing basic arithmetic functions (like ADD and SUBTRACT) and basic logic operations (like bit-wise AND, OR, and NOT)

    能够执行基本算术函数和基本逻辑运算

    • The ALU normally processes data elements of a fixed size referred to as the word length of the computer. The data elements are called words.

    ALU 通常处理固定大小的数据元素，称为计算机的字长。数据元素称为单词。

    在LC-3中，ALU process 16-bits words，has a word length of 16 bits

    • Almost all computers, have temporary storage for storing the result of A + B in order to avoid the much longer access time that would be necessary when it came time to multiply. The most common form of temporary storage is a set of registers

    几乎所有计算机都有临时存储器来存储 A + B 的结果，以避免乘法时需要更长的访问时间。最常见的临时存储形式是一组寄存器

    • Typically, the size of each register is identical to the size of values processed by the ALU; that is, they each contain one word. The LC-3 has eight registers (R0, R1, ... R7), each containing 16 bits.

    通常，每个寄存器的大小与 ALU 处理的值的大小相同；也就是说，它们每个都包含一个单词。 LC-3 有八个寄存器（R0、R1、...R7），每个寄存器包含 16 位。

    4.1.3 Input and Output¶

    4.1.4 Control Unit¶

    • It is the control unit that keeps track of both where we are within the process of executing the program and where we are in the process of executing each instruction.

    它跟踪我们在执行程序的过程中的位置以及我们在执行每条指令的过程中的位置

    • To keep track of which instruction is being executed, the control unit has an instruction register to contain that instruction.

    为了跟踪正在执行的指令，控制单元有一个指令寄存器来包含该指令。

    • To keep track of which instruction is to be processed next, the control unit has a register that contains the next instruction’s address.

    为了跟踪接下来要处理哪条指令，控制单元有一个寄存器，其中包含下一条指令的地址。

    1. program counter（PC）程序计数器 PC存放的是下一条指令的地址
    2. instruction register（IR）指令寄存器 IR存放当前正在执行的指令
    3. FSM：有限状态机根据IR，决定控制信号，并把控制信号传到其他部件（ALU）

    4.2 The LC-3：An example von Neumann Machine¶

    • Filled-in arrowheads denote data elements that flow along the corresponding paths. Not-filled-in arrowheads denote control signals that control the processing of the data elements

    实心箭头表示沿着相应路径流动的数据元素。未填充的箭头表示控制数据元素处理的控制信号

    • eg:the box labeled ALU in the processing unit processes two 16-bit values and produces a 16-bit result. The two sources and the result are all data, and are designated by filled-in arrowheads.The operation performed on those two 16-bit data elements (it is labeled ALUK) is part of the control—therefore, a not-filled-in arrowhead.

    处理单元中标有 ALU 的框处理两个 16 位值并产生 16 位结果。两个来源和结果都是数据，并用实心箭头表示。对这两个 16 位数据元素（标记为 ALUK）执行的操作是控制的一部分，因此是一个未填充的箭头

    1. Memory
    • MEMORY consists of the storage elements, along with the Memory Address Register (MAR) for addressing individual locations and
    • the Memory Data Register (MDR) for holding the contents of a memory location on its way to/from the storage

    • 存储器由存储元件以及用于寻址各个位置的存储器地址寄存器 (MAR) 组成和内存数据寄存器 (MDR)，用于在进出存储器的过程中保存内存位置的内容

    • 注意到，MAR contains 16 bits,说明LC-3的地址空间是\(2^{16}\) memory locations

    • MDR contains 16 bits，说明寻址能力是16 bits
    1. input and output

    • input and output consist of a keyboard and a minitor

    • The simplest keyboard requires two registers: a keyboard data register (KBDR) for holding the ASCII codes of keys struck and a keyboard status register (KBSR) for maintaining status information about the keys struck.

    一个键盘数据寄存器（keyboard data register），用于保存所敲击按键的ASCII码值，一个键盘状态寄存器（keyboard status register）维护有关所敲击按键的状态信息

    • The simplest monitor also requires two registers: a display data register (DDR) for holding the ASCII code of something to be displayed on the screen and a display status register (DSR) for maintaining associated status information.

    一个屏幕显示数据寄存器（DDR），用于保存要在屏幕上显示的内容的ASCII码，一个显示状态寄存器（DSR），用于维护相关的状态信息

    1. The processing unit
    • THE PROCESSING UNIT consists of a functional unit (ALU)that performs arithmetic and logic operations and eight registers (R0, ... R7) for storing temporary values that will be needed in the near future as operands for subsequent instructions.

    处理单元由一个执行算术和逻辑运算的功能单元（ALU）和八个寄存器（R0，... R7）组成，用于存储在不久的将来作为后续指令的操作数所需的临时值。

    • The LC-3 ALU can perform one arithmetic operation (addition) and two logical operations (bitwise AND and bitwise NOT).

    LC-3 ALU 可以执行一项算术运算（加法）和两项逻辑运算（按位 AND 和按位 NOT）

    1. The control unit
    • Its most important structure is thefinite state machine, which directs all the activity.
    • CLK specifies how long each clock cycle lasts
    • The instruction register (IR) is also an input to the finite state machine since the LC-3 instruction being processed determines what activities must be carried out.
    • The program counter (PC) is also a part of the control unit; it keeps track of the next instruction to be executed after the current instruction finishes

    4.3 Instruction processing 指令处理¶

    4.3.1 The instruction¶

    • The most basic unit of computer processing is the instruction. It is made up of two parts, the opcode (what the instruction does) and the operands (who it does it to!)

    opcode：操作码，指令做什么，operand：操作数，它对谁做

    • The LC-3 has threeoperate instructions: one arithmetic (ADD) and two logicals (AND and NOT).

    • Data movement instructions move information from the processing unit to and from memory and to and from input/output devices. (数据移动指令将信息从处理单元移入或移出存储器，移入或移出输入输出设备)The LC-3 has six data movement instructions.

    • Control instructions are necessary for altering the sequential processing of instructions(改变指令的顺序).

    normally the next instruction executed is the instruction contained in the next memory location.（执行的下一条指令是下一个存储位置中包含的指令） If a program consists of instructions 1,2,3,4...10 located in memory locations A, A+1, A+2, ...A+9（指令位于内存地址……）, normally the instructions would be executed in the sequence 1,2,3...10.

    关于LC-3的指令instruction

    1. An LC-3 instruction consists of 16 bits (one word), numbered from left to right, bit [15] to bit [0].
    2. Bits [15:12] contain the opcode. This means there are at most \(2^4\) distinct opcodes. Actually, we use only 15 of the possible four-bit codes. One is reserved for some future use.
    3. Bits [11:0] are used to figure out where the operands are.

    4.3.1.1 The add instruction¶

    • The ADD instruction is an operate instruction that requires three operands: two source operands (the numbers to be added) and one destination operand (where the sum is to be stored after the addition is performed).

    • ADD 指令要求至少两个源操作数之一包含在这些寄存器之一中，并且 ADD 的结果放入这八个寄存器之一中。

    • 格式：

    • 0001 表示opcode add [15:12]

    • Bits [11:9] identify the location to be used for storing the result(存储结果), in this case register6 (R6).

    • Bits [8:6] identify the register that contains one of the two source operands,in this case R2.

    • 两种表示方法的区别在于bit5是0还是1，

    • the first case, bit 5 is 0,signifying that the second source operand is in the register specified by bits [2:0], in this case R6.

      表示第二个源操作数位于位[2:0]指定的寄存器，本题为R6

    • In the second case,bit 5 is 1, signifying that the second source operand is formed by sign-extending(符号扩充) the integer in bits [4:0] to 16 bits. In this case, the second source operand is the positive integer 6.

    表示第二个源操作数是通过将位[4:0]中的整数符号扩展到16位而形成的。本题为正整数6

    4.3.1.2 The AND instruction¶

    • 格式和ADD 相同
    • opcode is 0101

    R3和00……00（16个）进行AND 操作，并将结果储存在R2中，达到初始化R2的目的

    4.3.1.3 The LD instruction¶

    • LD 表示 load，即到内存地址中取出数据并加载到寄存器中，“go to a particular memory location, read the value that is contained there, and store that value in one of the registers.”

    • LD 采用PC + offset 方式寻址

    • opcode is 0010

    • [11:9]表示用于存储的寄存器DR,需要将数据加载到寄存器中
    • Bits[8:0]表示offset偏移量，先将这9位bits拓展成16位，以二进制补码的形式求出值，将他加到PC（程序计数器，当前指令的位置）的当前位置中。
    • 上图中，指令告诉计算机将PC的内容加198形成内存位置的地址，并将该内存位置的内容加载到R2中

    注意点：此时的PC表示原先LD对应的PC再+1

    4.3.2 The instruction cycle 指令周期¶

    • fetch 获取
    • decode 解码
    • evaluate address 获取地址
    • fetch operands 获取操作数
    • execute 执行
    • store result 存储结果

    4.3.2.1 Fetch¶

    • The FETCH phase obtains the next instruction from memory and loads it into the instruction register (IR) of the control unit

    FETCH阶段从内存中获取下一条指令并将其加载到控制单元的指令寄存器（IR）中

    • The program counter（PC）contain the address of the next instruction to be processed

    步骤：

    • Step 1: Load the MAR with the contents of the PC, and simultaneously increment the PC.（one clock cycle）

    将PC的内容加载到MAR中，同时PC = PC + 1

    • Step 2: Interrogate memory, resulting in the instruction being placed in the MDR.（one or more clock cycle）

    访问内存，将指令放入MDR中

    • Step 3: Load the IR with the contents of the MDR.

    将MDR的内容加载到IR中（one clock cycle）

    4.3.2.2 decode¶

    FSM（finite state machine） ，根据IR存储的指令内容进行解码。

    确定操作码opcode，通过opcode前四位bits决定指令类型

    4.3.2.3 evaluate address¶

    计算我们要访问内存的地址。（这个阶段只会在我们需要访问内存时才 出现） e.g. LD

    LD 指令将存储在内存中的值加载到寄存器中。在该示例中，地址是通过将指令的位 [8:0] 符号扩展为 16 位并将该值添加到 PC 的当前内容来获得的。该计算是在评估地址阶段执行的。值得注意的是，并非所有指令都访问内存来加载或存储数据。例如，我们已经看到LC-3中的ADD和AND指令从寄存器或指令本身获取其源操作数，并将ADD或AND指令的结果存储在寄存器中。对于这些指令，不需要评估地址阶段。

    4.3.2.4 fetch operands¶

    取出源操作数（包括内存和寄存器堆中的数据）e.g. LD 在这个阶段 会从内存中取出数据，ADD 在这个阶段会从寄存器堆中取出数据。

    4.3.2.5 execute¶

    执行计算。 e.g. ADD 指令在这个阶段我们会加上源操作数。

    4.3.2.6 Store result¶

    把结果写回到目的地。 e.g. 比如 ADD 会把计算的结果写回到寄存器堆中，ST (store) 会把数据写回到内存中。

    虽然存在指令周期这个东西，但是并不一定所有的指令都要经过这六个过程。eg：ADD指令可以在一个时钟周期内，完成fetch operands，execute，store result

    4.3.3 Change the Sequence of Execution¶

    • control instruction——change the sequence of instruction execution.

    • bit[15:12] : opcode(0000表示BR的操作码)

    • bit[11:9] : 表示跳转的条件。（bit[11:9]分别对应NZP）

    • bit[8:0] : offset 偏移量，和LD一样需要符号扩充至16位，这样PC+offset 就是我们要跳转到的地址

    注意一下：，此处的PC是BR对应的原PC+1

    关于Conditional Code

    • CC（conditional codes）是三个1-bit的状态（NZP）。
    • N 表示 negative，Z表示zero，P表示positive。部分指令执行后，会根据执行结果更新NZP三个条件码。
    • BR根据CC来决定是否跳转。
    • 如果conditional code 全为1，则一定跳转
    • 如果conditional code 全为0，则一定不跳转
    • 其他情况，先看conditional code的1在哪一位，再看那个一对应的条件码是否为1，只要两者全为1，则满足跳转要求
    • 如果执行跳转，就用PC+offset 代替PC+1
    • 对于不执行跳转，仍然是PC+1

    4.3.4 Control of the instruction cycle¶

    1. Fetch阶段：需要三个时钟周期

    2. MAR is loaded with the contents of the PC，and the PC is incremented

      MAR 加载 PC（下一条指令的内存地址） 的内容，PC地址+1

      中间需要GatePC and LD.MAR.，GatePC connects the PC to the processor bus，GatePC将PC连接到总线，沿着总线到达Memory和MAR

      LD.MAR是MAR的write enable signal，需要在时钟周期结束之前将总线的内容（PC）加载到MAR上

    3. MDR读取MAR的数据

    4. 指令从MDR传输到指令寄存器（IR）。需要GateMDR和LD.IR

    5. Decode阶段：一个时钟周期

    6. 有限状态机，根据IR[15:12]中的特定操作码opcode，传输给ALU

    7. 可能还有根据IR[5]，判断是SR+SR，还是SR+imm（immediate）
    8. 因此还需要计算IR[4:0]的符号扩充至16位

    4.3.5 完整叙述ADD，LD，BR的全流程¶

    1. Fetch：

    MAR is loaded with the contents of the PC，and the PC is incremented（MAR 加载 PC（下一条指令的内存地址） 的内容，PC地址+1）中间需要GatePC and LD.MAR.，GatePC connects the PC to the processor bus，GatePC将PC连接到总线，沿着总线到达Memory和MAR

    LD.MAR是MAR的write enable signal，需要在时钟周期结束之前将总线的内容（PC）加载到MAR上

    MDR读取MAR的数据

    指令从MDR传输到指令寄存器（IR）。需要GateMDR和LD.IR

    1. Decode：

    IR出发，先传输到finite state machine，获得bits[15:12],将opcode操作码转化后传到ALUK。

    分支检查bit[5],判断add的类型。

    分支 sign-extending（SEXT），将bits[4:0]进行符号扩充，作为第二个加数

    1. Evaluate address：

    add都是从寄存器取值，不需要内存，这一步不发挥作用

    1. Fetch operands：

    从SR1和SR2中取出值，关于SR2的值需要经过多路选择器（SR2MUX），有之前的bit[5]决定是哪个input

    1. Execute：

    SR2MUX和SR1OUT出来的值到ALU，ALUK决定计算的类型ADD，进行计算

    1. Store result：

    ALU出来的值经过ALUgate到达processing bus（总线）往上走，到REG FILE，最后保存在DR中

    4.4 Our first program：A Multiplication Algorithm¶

    • 乘法展开就是加法
    • Let us assume that memory location x3007, abbreviated M[x3007], contains the value 5, and M[x3008] contains the value 4. We start by copying the two values from memory to the two registers R1 and R2. We are going to accumulate the results of the additions in R3, so we initialize R3 to 0. Then we add 5 to R3, and subtract 1 from R2 so we will know how many more times we will need to add 5 to R3. We keep doing this (adding 5 to R3 and subtracting 1 from R2) until R2 contains the value 0. That tells us that we have added 5 to R3 four times and we are done, so we HALT the computer. R3 contains the value 20, the result of our multiplication.
