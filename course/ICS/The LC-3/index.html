    Chapter 5 | The LC-3¶

    5.1 The ISA：Overview¶

    1. Organization
    • address space \(2^{16}\)
    • addressability : 16 bits
    1. Registers

    • The most common type of temporary storage locations, and the one used in the LC-3, is a set of registers.

    • Each register in the set is called a general purpose register (GPR).

    • The number of bits stored in each register is usually one word. In LC-3 means 16 bits

    1. The instruction set

    • The instruction set is defined by its set of opcodes, data types, and addressing modes.

    • The addressing modes determine where the operands are located.

      e.g. ADD instruction have two addressing modes：register mode and immediate mode

    • The data type is the representation of the operands in 0s and 1s.

    1. Opcodes
    • The x86 ISA has more than 200 opcodes.
    • The LC-3 ISA has 15 instructions, each identified by its unique opcode

    1. Data types

    2. Addressing modes

    • An operand can generally be found in one of three places: in memory, in a register, or as a part of the instruction.

    • If the operand is a part of the instruction, we refer to it as a literal or as an immediate operand.

    • LC-3支持五种addressing modes：immediate，register，PC-relative，indirect，Base+offset
    1. Condition Codes

    • The three single-bit registers are called N, Z, and P, corresponding to their meaning: negative, zero, and positive.

    • The set of three single-bit registers are referred to as condition codes because the condition of those bits are used to change the sequence of execution of the instructions in a computer program

    用于更改计算机程序中指令的执行顺序

    5.2 Operate Instruction¶

    5.2.1 ADD AND NOT（arithmetic and logic）¶

    • The LC-3 has three operate instructions: ADD, AND, and NOT.
    1. NOT

    • opcode = 1001

    • The NOT instruction bit-wise complements a 16-bit source operand and stores the result in a destination registe

      NOT 指令按位求补 16 位源操作数并将结果存储在目标寄存器中。

    • Bits[11:9]表示目标寄存器DR（destination register）

    • Bits[8:6]表示源寄存器（source register）

    • Bits[5:0]全部都是1

    If R5 initially contains 0101000011110000, after executing the following instruction:R3 will contain 1010111100001111

    NOT is a unary operation.NOT是一元运算

    ADD and AND 都是二元运算，它们需要两个 16 位源操作数

    但是两者都采用register addressing mode（寄存器寻址模式）

    5.2.2 Immediates¶

    提到的两种寻址模式：register mode ， immediate addressing mode

    If bit [5] is 1, the second source operand is contained within the instruction. In that case the second source operand is obtained by sign-extending bits [4:0] to 16 bits before performing the ADD or AND.

    如果位 [5] 为 1，则第二个源操作数包含在指令内。在这种情况下，在执行 ADD 或 AND 之前，通过将位 [4:0] 符号扩展至 16 位来获得第二个源操作数。

    • 相反数的二进制补码是逐位取反再加一
    • 第一步执行逐位取反，并保存在寄存器R1中
    • 第二步，进行加一操作，并将结果保存在R2
    • 第三步，进行A加（-B）操作，A在R0中，-B在R2中

    5.2.3 The LEA instruction¶

    • LEA：it simply loads a register with an address. 将地址加载进寄存器

    • opcode = 1110

    • LEA (opcode = 1110) loads the register specified by bits [11:9] of the instruction with the value formed by adding the incremented program counter to the sign-extended bits [8:0] of the instruction.

    LEA（操作码= 1110）将递增的程序计数器与指令的符号扩展位[8:0]相加而形成的值（PC + offset）加载到指令的位[11:9]指定的寄存器中。

    • 和LD的不同之处在于，LEA不访问内存，只是单纯地将地址加载到寄存器。而LD需要从内存中取出数据并加载到寄存器中

    PC = PC + 1 = x4019,offset = -3,PC + offset = x4016

    5.3 Data Movement instruction¶

    • The process of moving information from memory to a register is called a load,

    • and the process of moving information from a register to memory is called a store.

    LC-3 contains six instructions that move information：LD，LDR，LDI，ST，STR，STI

    需要注意的是只有 LD，LDR，LDI 会设置 CC 条件码，store 类指令不会。如果是 load 类指令，我们会根据最后从内存中加载出来，并且要被放入寄存器的值更新 CC。

    • The source is the data to be moved

    • The destination is the location where it is moved to

    • Bits [11:9] specify one of these operands, the register. If the instruction is a load, DR refers to the destination general purpose register that will contain the value after it is read from memory (at the completion of the instruction cycle).

    位 [11:9] 指定这些操作数之一，即寄存器。如果指令是加载，则 DR 指的是目标通用寄存器，该寄存器将包含从内存读取后（指令周期完成时）的值。

    • If the instruction is a store, SR refers to the register that contains the value that will be written to memory.

    如果指令是存储，则 SR 指的是包含将写入内存的值的寄存器

    5.3.1 PC-Relative Mode([PC+1-2^8^,PC+2^8^])¶

    LD（opcode = 0010）和ST（opcode = 0011）都是PC-relative addressing mode

    之所以如此命名，是因为指令的位 [8:0] 指定了相对于 PC 的偏移量。内存地址是通过将位[8:0]符号扩展至16位并将结果添加到递增的PC来计算的。递增的 PC 是 FETCH 阶段之后（即 PC 递增之后）程序计数器的内容。

    • 对于LD，计算出来的结果（PC+offset）是指定要访问的内存地址，它的内容将被保存在目标寄存器DR（由Bits[11:9]）决定。
    • 对于ST，指令将位[11:9]指定的寄存器的内容写入地址为PC=offset的内存地址

    注意点：此处计算偏移量，首先进行符号补位，转换为xFFAF，再与PC+1 = x4019相加得到x3FC8

    1. 关于LD，需要取出指定地址的内容数据并存入寄存器。所以，首先计算出该指定地址PC+offset，并存入MAR
    2. 找到MAR对应的MDR中的数据
    3. 将MDR的value存入寄存器中

    还会发现，offset是bits[8:0]的二进制补码对应的值，取值范围是[-256,255],如果需要偏移量大于这个范围，就要使用其他寻址模式

    PC相对的寻址方式能访问哪个范围内的数据？

    \([PC+1-2^8，PC+1+2^8-1=PC+2^8]\)

    5.3.2 Indirect Mode（无范围限制）¶

    LDI（opcode = 1010）和STI（opcode = 1011）specify the indirect addressing mode

    间接寻址，指令格式和 LD 相同，只是需要访问两次内存：首先计算出PC+offset，随后访问内存取出值，然后将刚刚取出的值作为地址再次访问内存，最后取出的值加载到寄存器中。

    然而，该地址不是要加载或存储的操作数的地址，而是要加载或存储的操作数的地址的地址。因此称为间接

    PC+offset = x4A1C + xFFCC = x49EB,发现49EB中存放着的是一个地址x2110，从而将x2110的内容存入目标就寄存器中

    1. 计算PC + offset，并将所得地址x49EB存入MAR中
    2. x49EB的内容x2110被存入目标寄存器R3中
    3. x2110不是operand，而是operand的地址，将x2110存放到MAR中
    4. memory is read，MDR again loaded。MDR is loaded with the contents of x2110
    5. 将MDR的数据存入目标寄存器

    Indirect可以访问任何范围的地址

    5.3.3 Base+offset Mode（无范围限制）¶

    LDR (opcode = 0110) and STR (opcode = 0111) specify the Base+offset addressing mode.

    • Base + offset: The address of the operand is obtained by adding a sign-extended six-bit offset to a base register.
    • bits[5:0]表示偏移量offset
    • bits[8:6]表示基址寄存器base register
    • bits[11:9]表示目标寄存器destination register

    1. R2 的value x2345和offset x001D进行相加得到x2362，并将x2362存入MAR中
    2. 读取内存，x2362的内容存入MDR中
    3. 将MDR中存放的内容存入R1

    Base+offset，可以访问任何范围内的地址，只要在基址寄存器中改变数据即可

    5.3.4 An Example¶

    1. PC = x30F6, opcode = 1110，说明是LEA（单纯将地址放入寄存器中），PC + offset = x30F7 + xFFFD = x30F4,即R1存放 x30F4
    2. PC = x30F7，opcode = 0001，说明是ADD，先计算后五位bits的immediate，进行符号扩充，得到imm = \(（14）_{10} = E_{16}\)。SR1 + imm = x30F4 + x000E = x3102，从而R2 存放 x3102
    3. PC = x30F8，opcode = 0011，说明是ST，采用PC-relative addressing mode，PC + offset = x30F9 + xFFFB = x30F4。从而R2 存放的是地址x30F4
    4. PC = x30F9，opcode = 0101，说明是AND，对寄存器R2进行初始化，R2存放0
    5. PC = x301A，opcode = 0001，说明是ADD，R2 = R2 + imm = 5
    6. PC = x301B，opcode = 0111，说明是STR（use base+offset addressing mode，基址寄存器 + 偏移量），基址寄存器是R1，R1存放着x30F4，偏移量offset = 14 = \(E_{16}\),则基址寄存器+偏移量 = x30F4 + x000E = x3102，将寄存器R2的内容value = 5写入内容x3102，即M[x3102] = 5
    7. PC = x30FC，opcode = 1010，表示LDI（LD + indirect addressing mode，存放地址的地址 of operand），PC + offset = x30FD + xFFF7 = x30F4，x30F4存放着operand 的地址，又M[30F4] = x3102，则x3102是operand = 地址，M[x3102] = 5,说明operand操作数是5，将value = 5 写入R3中

    5.4 Control Instructions¶

    The LC-3 has five opcodes that enable the sequential execution flow to be broken: conditional branch, unconditional jump, subroutine call (sometimes called function), TRAP, and RTI (Return from Trap or Interrupt).

    5.4.1 Conditional Reanches¶

    Based on the execution of previous instructions in the program, the conditional branch’s EXECUTE phase either does nothing or it loads the PC with the address of the instruction it wishes to execute next.

    根据程序中先前指令的执行，条件分支的 EXECUTE 阶段要么不执行任何操作，要么向 PC 加载它希望接下来执行的指令的地址。

    During the EXECUTE phase of the BR instruction cycle, the processor examines the condition codes whose associated bits in the instruction, bits [11:9], are 1. Note the lower case n, z, and p in bits [11:9] of the BR instruction format shown above.

    • If bit [11] is 1, condition code N is examined. If bit [10] is 1,condition code Z is examined. If bit [9] is 1, condition code P is examined.

    • If any of bits [11:9] are 0, the associated condition codes are not examined.

    • If any of the condition codes that are examined are set (i.e., equal to 1), then the PC is loaded with the address obtained in the EVALUATE ADDRESS phase.

    如果设置了所检查的任何条件代码（即等于 1），则 PC 将加载在 EVALUATE ADDRESS 阶段获得的地址。

    我们跳转的条件为n * N+ z * Z + p * P 为真>0

    • If none of the condition codes that are examined are set, the incremented PC is left unchanged, and the next sequential instruction will be fetched at the start of the next instruction cycle.

    如果没有设置所检查的条件代码，则递增的 PC 保持不变，并且将在下一个指令周期开始时获取下一个顺序指令。

    • Each of the three AND gates corresponds to one of the three condition codes.The output of the AND gate is 1 if the corresponding condition code is 1

    三个与门中的每一个对应于三个条件代码之一。如果相应的条件码为1，则与门的输出为1

    • If any of the three AND gates have an output 1, the OR gate has an output 1, indicating that the sequential instruction flow should be broken, and the PC should be loaded with the address evaluated during the EVALUATE ADDRESS phase of the instruction cycle.

    如果三个 AND 门中的任何一个具有输出 1，则 OR 门具有输出 1，表明顺序指令流应被中断，并且 PC 应加载在指令周期的 EVALUATE ADDRESS 阶段期间评估的地址

    In the case of the conditional branch instruction at x4027, the answer is yes, and the PC is loaded with x4101, replacing x4028, which had been loaded into the PC during the FETCH phase of the BR instruction.

    若条件码conditional code 都为1，则一定跳转

    若条件码conditional code 都为0，则一定不跳转，执行PC+1

    此处的第一步是为了获取CC，ADD操作后CC会发生改变，针对的R0的值。此处R0+R0不会改变R0的正负性

    5.4.2 Two methods of Loop Control¶

    The BR instruction at the end of the sequence controls the number of times the loop body is executed

    BR指令控制循环语句进行的次数

    5.4.2.1 Method1：Loop Control with a Counter¶

    Suppose we know that the 12 locations x3100 to x310B contain integers, and we wish to compute the sum of these 12 integers.

    问题：假设我们知道 x3100 到 x310B 的 12 个位置包含整数，并且我们希望计算这 12 个整数的总和。

    1. initialize the variables.初始化变量

    • the address of the next integer to be added (assigned to R1)

      R1 = x3100

    • the running sum (assigned to R3)

      R3 = 0

    • the number of integers left to be added (assigned to R2).

      R2 = 12

    1. The program repeats the process of loading into R4 one of the 12 integers and adding it to R3.

    2. Each time we perform the ADD, we increment R1 so it will point to (i.e., contain the address of) the next number to be added and decrement R2 so we will know how many numbers still need to be added.

    3. When R2 becomes zero, the Z condition code is set, and we can detect that we are done.

    • PC = x3000，opcode = 1110，LEA（将地址x3100写入寄存器R1）。PC + offset = x3001 + x00FF = x3100，确定offset = x00FF，转换为二进制，011111111

    • PC = x3001，opcode = 0101，AND（R3初始化）。

    • PC = x3002，opcode = 0101，AND（R2初始化）。

    • PC = x3003，opcode = 0001，ADD（对R2 = R2 + 12）

    • PC = x3004，opcode = 0000，BR（Bit[10] = 1,检查Z的条件代码，如果对应的R2的值为0，则跳转到x300A=x3005+x0005）

    • PC = x3005，opcode = 0110，LD（Base+offset Mode，将R1中存放的地址对应的value 存放到R4）

    • PC = x3006，opcode = 0001，ADD（将R3和R4进行相加，并将结果存入R3）

    • PC = x3007，opcode = 0001，ADD（R1+1）R1 will point to the next location in memory containing an integer to be added.将指向内存中包含要相加的整数的下一个位置。

    • PC = x3008，opcode = 0001，ADD（R2- 1）

    • PC = x3009，opcode = 0000，BR（NEP= 111，unconditional branch，无条件跳转，PC+offset = x300A+xFFFA = x3004）.

    so the next instruction to be executed (the conditional branch at x3004) will be based on the instruction executed at x3008.

    The conditional branch instruction at x3004 follows the instruction at x3009, which does not affect condition codes, which in turn follows the instruction at x3008.

    因为x3009处的指令不影响条件代码

    因此下一条要执行的指令（x3004 处的条件分支）将基于 x3008 处执行的指令。即根据R2的值进行判断是否跳转

    • The conditional branch instruction causes the execution sequence to follow: x3000, x3001, x3002, x3003, x3004, x3005, x3006, x3007, x3008, x3009, x3004, x3005, x3006, x3007, x3008, x3009, x3004, x3005, and so on. The loop body consists of the instructions at x3005 to x3009. When the value in R2 becomes 0, the PC is loaded with x300A, and the program continues at x300A with its next activity.

    • 后续只需在x300A添加halt指令停止即可

    5.4.2.2 Method2：Loop Control with a Sentinel¶

    前一种方法采用计数器，已知总共需要执行的次数为12，只需在每次执行完循环后，counter-1，并且看counter是否为0即可

    A second method for controlling the number of executions of a loop is to use a sentinel.

    控制循环执行次数的第二种方法是使用哨兵

    This method is particularly effective if we do not know ahead of time how many iterations we will want to perform.

    如果我们事先不知道要执行多少次迭代，则此方法特别有效。

    假设我们知道存储在 x3100 到 x310B 位置的值都是正数。然后我们可以使用任何负数作为哨兵。

    假设存储在内存地址x310C处的哨兵是-1.

    • PC = x3000，opcode = 1110，LEA（将地址x3100写入寄存器R1）。PC + offset = x3001 + x00FF = x3100，确定offset = x00FF，转换为二进制，011111111

    • PC = x3001，opcode = 0101，AND（R3初始化）。

    • PC = x3002，opcode = 0110，LD（Base+offset Mode，将R1中存放的地址对应的value 存放到R4）

    • PC = x3003，opcode = 0000，BR（n，x3008）。PC + offset = x3004 + x0004 = x3008.

    一旦（哨兵）为负数成立，就跳转到x3008指令halt

    • PC = x3004，opcode = 0001，ADD（将R3和R4进行相加，并将结果存入R3）

    • PC = x3005，opcode = 0001，ADD（R1+1）R1 will point to the next location in memory containing an integer to be added.将指向内存中包含要相加的整数的下一个位置。

    • PC = x3006，opcode = 0110，LD（Base+offset Mode，将R1中存放的地址对应的value 存放到R4）

    • PC = x3007，opcode = 0000，BR（NEP= 111，unconditional branch，无条件跳转，PC+offset = x3008+xFFFB = x3003）.

    5.4.3 The JMP Instruction¶

    • 普通的BR，offset有范围限制，所以跳转的范围也受限制

    • The JMP instruction loads the PC with the contents of the register specified by bits [8:6] of the instruction

    JMP 指令将指令的位 [8:6] 指定的寄存器内容加载到 PC

    即JMP直接跳到BaseR所存储的值处，可以跳转到任何地方，无范围限制

    R2 contains the value x6600（对应于指定寄存器的内容，需要加载到PC，即下一条执行）, and the PC contains x4000, then the instruction at x4000 (the JMP instruction) will be executed, followed by the instruction located at x6600. Since registers contain 16 bits (the full address space of memory), the JMP instruction has no limitation on where the next instruction to be executed must reside.

    R2包含值x6600，而PC包含x4000，则将执行x4000处的指令（JMP指令），然后执行位于x6600处的指令。由于寄存器包含 16 位（存储器的完整地址空间），因此 JMP 指令对于要执行的下一条指令必须驻留在何处没有限制。

    5.4.4 The TRAP Instruction¶

    The TRAP (opcode = 1111)instruction changes the PC to a memory address that is part of the operating system so that the operating system will perform some task on behalf of the program that is executing.

    TRAP（操作码 = 1111）指令将 PC 更改为操作系统一部分的内存地址，以便操作系统将代表正在执行的程序执行某些任务。

    • bit[15:12] = 1111
    • bit[11:8] = 0000,默认为0
    • bits[7:0] : trapvector

    TRAP执行的是调用系统的服务（从键盘获取输入，输出到终端，终止程序halt……）

    • Input a character from the keboard —— trapvector = x23
    • Output a character to the monitor —— trapvector = x21
    • Halt the program —— trapvector = x25
    • 对于 trap 指令，我们会把 trapvector 零扩充为 16 位作为地址，取出对应地址的值。这个值就是 trap 处理程序的地址，接下来我们跳到处理程序的地方即可。

    5.5 Another Example：Counting Occurrence of a Character计算字符出现的次数¶

    1. R0 will hold the character that is being counted R0

    R0将保存正在计数的字符

    1. R1 will hold, in turn, each character that we get from the file being examined

    R1 将依次保存我们从正在检查的文件中获得的每个字符

    1. R2 will keep track of the number of occurrences;

    用于记录出现的次数

    1. R3 will point to the next character in the file that is being examined. We refer to it as a pointer since it points to (i.e., contains the address of) the location where the next character of the file that we wish to examine resides.

    R3 将指向正在检查的文件中的下一个字符。我们将其称为指针，因为它指向（即包含其地址）我们希望检查的文件的下一个字符所在的位置。

    1. Processing each character requires one iteration of a loop. We will use the sentinel method, using the ASCII code for EOT (End of Transmission) (00000100) as the sentinel.

    处理每个字符需要循环一次迭代。我们将使用哨兵方法，使用EOT（传输结束）的ASCII码（00000100）作为哨兵。

    1. 在循环的每一次迭代中，首先将R1的内容和EOT的ASCII码进行比较，如果相等则退出循环，程序进入最后一步。如果不相等，则R1(当前正在检查的字符)和R0（从键盘输入的字符）进行比较，如果两者相等，则R2+1，不相等则R2保持不变。后续R3指针+1，将下一个字符加载到R1，返回到检查R1和哨兵是否相等

    2. 当到达文件末尾后，所有的字符都已被检查，并且计数以二进制形式包含在R2中

    • PC = x3000，opcode = 0101，AND，初始化R2，R2 will keep track of the number of occurrences; 用于记录出现的次数

    • PC = x3001，opcode = 0010，LD，loads the starting address of the file to be examined into R3.

    • PC = x3002，opcode = 1111，TRAP，requests the operating system to perform a service call on behalf of this program. The function requested, as identified by the eight-bit trapvector 00100011 (i.e., x23), is to load into R0 the ASCII code of the next character typed on the keyboard. 请求操作系统代表该程序执行服务调用。所请求的功能由八位陷阱向量 00100011（即 x23）标识，是将键盘上键入的下一个字符的 ASCII 代码加载到 R0 中

    • PC = x3003，opcode = 0110，LDR（DR + Base），offset = 0，loads the character pointed to by R3 into R1

    • PC = x3004，opcode = 0001，ADD，subtracting 4 (the ASCII code for EOT) from R1 and storing it in R4. 从 R1 中减去 4（EOT 的 ASCII 代码）并将其存储到 R4 中。If the result is zero, the end of the file has been reached, and it is time to output the count.如果结果为零，则已到达文件末尾，是时候输出计数了。

    • PC = x3005，opcode = 0000，BR，If the result is zero, the end of the file has been reached, and it is time to output the count.如果结果为零，则已到达文件末尾，是时候输出计数了。跳转到x300E

    • If R4 is not equal to zero, the character in R1 is legitimate and must be examined. The sequence of instructions at locations x3006, x3007, and x3008 determines whether the contents of R1 and R0 are identical. Taken together, the three instructions compute R0 − R1 This produces all zeros only if the bit patterns of R1 and R0 are identical.

    如果R4不等于0，则进入下一条指令，后续判断R1和R0是否相等。x3006,x3007,x3008处的指令合在一起判断R0和R1是否相等。

    先x3006，对R1进行逐位取反，再x3007，对R1+1操作，得到R1的相反数，最后x3008，ADD，R1+R0保存在R1中

    • PC = x3009，opcode = 0000，BR，判断R1是正数负数或者0.如果R1是0，则不跳转，进入下一条R2 = R2+1，计数加一；如果R1不等于0，则跳转到x300B，R3=R3+1，x300B 处的指令递增 R3，因此它将指向正在检查的文件中的下一个字符

    • PC = x300C，opcode = 0110，LDR（DR + Base），offset = 0，loads the character pointed to by R3 into R1，开始检测新的字符

    • PC = x300D，opcode = 0000，BR，跳转到x3004，新的一轮开始

    • PC = x300E，opcode = 0010，LD，此时，已经检测到哨兵EOT，将 00110000 加载到 R0 中

    • PC = x300F，opcode = 0001，ADD，x300F 处的指令将计数添加到 R0 中，这会将计数的二进制表示形式（在 R2 中）转换为计数的 ASCII 表示形式（在 R0 中）。

    • PC = x3010，opcode = 1111，TRAP，调用操作系统的 TRAP，将 R0 的内容输出到监视器。

    • PC = x3011，opcode = 1111，TRAP，调用 TRAP 指令来终止程序
