    Chapter 7 | Assembly Language¶

    • Assembly language is a low-lever language汇编语言是低级语言

    • Each assembly language instruction usually specifies a single instruction in the ISA. 每一种汇编语言指令，通常指定ISA的一条指令

    • Unlike high-level languages, which are usually ISA independent, low-level languages are very much ISA dependent. In fact, it is usually the case that each ISA has only one assembly language.

    高级语言不依赖于ISA，低级语言非常依赖于ISA，通常每一个ISA只有一种汇编语言

    7.2 Instructions¶

    Instead of an instruction being 16 0s and 1s, as is the case in the LC-3 ISA, an instruction in assembly language consists of four parts, as follows: Two of the parts (Label and Comment) are optional.

    7.2.1 Opcodes and Operands¶

    • Opcode(the thing the instruction is to do) and Operands（the thing it is supposed to do） are mandatory.

    操作码（指令要做的事情）和操作数（它应该做的事情）是强制性要求具备的

    1. 对于ADD，require three operands（Two sources to obtain the number to be added，and one destination to designate where the result is to be stored） $$ ADD DR,SR1,SR2 $$

    2. 对于LD，require two operands（the memory location from which the value is to be read and the destination register that is to contain the value after the instruction finishes execution要从中读取值的内存位置以及指令完成执行后要包含该值的目标寄存器)

    在LC-3的汇编语言中，我们用Label 代替内存位置，无需记住16位地址 $$ LD DR ,Label $$

    1. 对于AND，如果AND一个immediate立即数 $$ AND DR,SR,井号 number\ AND R3,R3,井号 0 ;clear R3. $$

    We use # for decimal, x for hexadecimal, and b for binary

    7.2.2 Labels¶

    Labels are symbolic names that are used to identify memory locations that are referred to explicitly in the program

    标签是符号名称，用于标识程序中显式引用的内存位置

    There are two reasons for explicitly referring to a memory location.显式引用内存位置有两个原因

    1. The location is the target of a branch instruction (e.g., AGAIN in line 0C). That is, the label AGAIN identifies the location of the instruction that will be executed next if the branch is taken.

    该位置是分支指令的目标（例如，0C 行中的 AGAIN）。也就是说，标签 AGAIN 标识了如果发生分支则接下来将执行的指令的位置。

    1. The location contains a value that is loaded or stored (e.g., NUMBER in line 12, and SIX in line 13).

    该位置包含已加载或存储的值

    Label 标签是用来标识内存地址的，相当于地址的一个别名

    7.2.3 Comments¶

    ;分号表示该行的其余部分是注释，将被汇编器忽略。

    7.3 Pseudo-Ops(Assembler Directives)伪操作（汇编指令）¶

    The LC-3 assembly language contains five pseudo-ops that we will find useful in our assembly language programming: .ORIG, .FILL, .BLKW, .STRINGZ, and .END. All are easily recognizable by the dot as their first character.

    LC-3 汇编语言包含五个伪操作，我们会发现它们在汇编语言编程中非常有用：.ORIG、.FILL、.BLKW、.STRINGZ 和 .END。所有这些都可以通过点作为其第一个字符轻松识别。

    1. .ORIG

    .ORIG tells the assembler where in memory to place the LC-3 program.

    .ORIG 告诉汇编器在内存中放置 LC-3 程序的位置。

    1. ,FILL

    .FILL tells the assembler to set aside **the next location** in the program and **initialize it with the value of the operand**.The value can be either a number or a label.

    .FILL 告诉汇编器预留程序中的下一个位置并使用操作数的值对其进行初始化。该值可以是数字或标签。

    .FILL 会把 operand 的值填充到当前地址（.FILL 的地址）

    1. ,BLKW

    .BLKW tells the assembler to set aside some number of sequential memory locations (i.e., a BLocK of Words) in the program.

    .BLKW 告诉汇编器在程序中预留一定数量的连续内存位置（即字块）。

    假设当前地址为 x3000 那么 .BLKW 10 就会将从 x3000 开始的是个地址全部留出来（从 x3000到 x3009），下一条指令的地址为 x300A。

    1. STRINGZ

    .STRINGZ tells the assembler to initialize a sequence of n +1 memory locations.The argument is a sequence of n characters inside double quotation marks. The first n words of memory are initialized with the zero-extended ASCII codes of the corresponding characters in the string. The final word of memory is initialized to \0. The last word, containing x0000, provides a convenient sentinel for processing the string of ASCII codes.

    .STRINGZ 告诉汇编器初始化 n+1 个内存位置的序列。参数是双引号内的 n 个字符的序列。内存的前 n 个字用字符串中相应字符的零扩展 ASCII 代码进行初始化。存储器的最后一个字被初始化为0。最后一个字包含x0000，为处理ASCII 代码串提供了方便的标记。

    n个字符zero-extending ASCII，最后字符用0结尾x0000

    1. .END

    .END tells the assembler it has reached the end of the programand need not even look at anything after it.. That is, any characters that come after .END will not be processed by the assembler.

    .END 告诉汇编器它已经到达程序的末尾，甚至不需要查看后面的任何内容。也就是说，.END 之后的任何字符都不会被汇编器处理。

    1. .END并不会结束程序的运行，只是一个分隔符
    2. 实际上一个程序可以有多个.END。每一组.ORIG和.END表示程序的某一段

    example: Count Character

    The ASCII codes for the decimal digits 0 to 9 (0000 to 1001) are x30 to x39. The conversion from binary to ASCII is done simply by adding x30 to the binary value of the decimal digit

    十进制数字 0 到 9（0000 到 1001）的 ASCII 代码为 x30 到 x39。从二进制到 ASCII 的转换只需将十进制数字的二进制值加上 x30 即可完成

    Line 2D shows the label ASCII used to identify the memory location containing x0030.

    7.4 The Assembly Process¶

    在执行 LC-3 汇编语言程序之前，必须首先将其翻译为机器语言程序，即其中每条指令都位于 LC-3 ISA 中的程序。 LC-3 汇编器的工作就是执行该转换。

    命令是 assemble，并且它需要汇编语言程序的文件名作为参数。

    assemble solution1.asm outfile produce the file outfile

    There is in general a one-to-one correspondence between instructions in an assembly language program and instructions in the final machine language program.

    一般来说，汇编语言程序中的指令与最终机器语言程序中的指令之间是一一对应的。

    The assembly process is done in two complete passes,整个汇编语言程序的汇编过程分两遍完成

    The objective of the first pass is to identify the actual binary addresses corresponding to the symbolic names (or labels). This set of correspondences is known as the symbol table.

    第一遍的目标是识别与符号名称（或标签）相对应的实际二进制地址。这组对应关系称为符号表。

    In pass 1, we construct the symbol table.

    在第 1 遍中，我们构建了符号表。

    In pass 2, we translate the individual assembly language instructions into their corresponding machine language instructions.

    在第 2 遍中，我们将各个汇编语言指令翻译为其相应的机器语言指令。

    7.4.1 The First Pass:Creating the Symbol Table¶

    the symbol table is simply a correspondence of symbolic names with their 16-bit memory addresses

    符号表只是符号名称与其 16 位内存地址的对应关系

    The assembler examines each instruction in sequence and increments the LC once for each assembly language instruction. If the instruction examined contains a label, a symbol table entry is made for that label, specifying the current contents of LC as its address. The first pass terminates when the .END pseudo-op is reached.

    汇编器按顺序检查每条指令，并对每条汇编语言指令将 LC 递增一次。如果检查的指令包含标签，则会为该标签创建一个符号表条目，并将 LC 的当前内容指定为其地址。当到达 .END 伪操作时，第一遍终止。

    地址表示 Label 所在PC的地址

    7.3.4 The Second Pass:Generating the Machine Language Program¶

    在每一行，汇编语言指令都被翻译成 LC-3 机器语言指令（01代码）

    当汇编器到达第 0C 行时，它可以完整地汇编该指令，因为它知道 PTR 对应于 x3013。该指令为LD，操作码编码为0010。目的寄存器（DR）为R3，即011。LD 指令中唯一剩下要做的部分是 PCoffset。

    为了使用 LD 指令，加载源（在本例中为标签为 PTR 的地址）必须距 LD 指令本身不超过 +256 或 −255 个内存位置。如果 PTR 的地址大于 LC+1+255 或小于 LC+1−256，则偏移量将无法容纳在指令的位 [8:0] 中。

    如果超出了范围应该怎么办，使用Label，本质上是一个地址
