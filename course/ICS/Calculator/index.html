    Chapter 10 | Calculator¶

    10.1 Data Type Conversion¶

    已知的数据类型：

    1. unsigned integers for address arithmetic（用于地址运算的无符号整数）
    2. 2‘s complement integers for integer arithmetic（用于整数运算的二进制补码整数）
    3. 16-bit binary strings for logical operations（用于逻辑运算的16位二进制字符串）
    4. floating point numbers for scientific computation（用于科学计算的浮点数）
    5. ASCII codes for interaction with output and input（用于输入与输出设备交互的ASCII代码）

    10.1.1 Example: A Bogus Program:2+3=e（虚假程序）¶

    what will happen？

    假设通过键盘输入的第一个数字是 2，通过键盘输入的第二个数字是 3。在程序终止之前，显示器上会显示什么？输入 2 后加载到 R0 中的值是 2 的 ASCII 代码，即 x0032。当输入 3 时，3 的 ASCII 代码（即 x0033）被加载到 R0 中（当然，在 2 的 ASCII 代码移至 R1 之后）。因此，ADD 指令将两个二进制字符串 x0032 和 x0033 相加，生成 x0065。当该值显示在监视器上时，它被视为 ASCII 代码。由于 x0065 是小写 e 的 ASCII 代码，因此显示器上会显示小写 e。

    10.1.2 Input Data（ASCII to Binary）¶

    图 10.2 显示了三位十进制整数 295 的 ASCII 表示形式，它作为 ASCII 字符串存储在三个连续的 LC-3 内存位置中，从 ASCIIBUFF 开始。 R1 包含正整数的位数。我们的 ASCII 到二进制子例程将整数限制在 0 到 999 的范围内。

    ASCIIBUFF is the address of the first memory location of a sequence of four memory locations that we have allocated (a) to store the ASCII codes of decimal digits entered from the keyboard, and (b) to store the ASCII codes corresponding to the result of arithmetic operations in preparation for writing it (the result) to the monitor.

    ASCIIBUFF 是我们分配的四个内存位置序列中的第一个内存位置的地址，（a）用于存储从键盘输入的十进制数字的 ASCII 代码，（b）存储与算术运算结果相对应的 ASCII 代码，以准备将其（结果）写入监视器。

    如何将十进制转化为二进制？

    首先获取每一位数字（0-9，只需四个bit便可以表示），去掉最后四位以外的所有位，将ASCII码转化为二进制代码，在0 - 9的二进制表中进行检索。由于我们转化的整数最多只有三位，所以有三张表，一张是百位数字，一张是十位数字，一张是个位数字。

    e.g.百位表中索引6 的条目值是#600，其二进制形式是0000001001011000.

    10.2 Arithmetic Using a Stack¶

    10.2.1 The stack as Temporary Storage¶

    1. LC-3是three-address machines（三地址机）在LC-3中，ADD指令需要三个寄存器发挥作用，DR目标寄存器，和SR1，SR2两个源寄存器

    2. 同时也存在着双地址机，``x86的ISA中ADD指令是 - ADD EAX，EBX 其中EAX和EBX是x86ISA中八个通用寄存器中的两个。EAX既充当源操作数的位置，也充当目标操作数的位置。对于双地址机器，源寄存器的结果为被覆盖

    3. 还有一些 ISA 根本不使用通用寄存器来存储源操作数或操作指令的结果。其中最常见的称为堆栈机，因为堆栈用于临时存储。 - 源操作数是通过从堆栈中弹出顶部两个元素来获得的。结果（即目标操作数）随后被推入堆栈。 - 由于计算机总是从堆栈中弹出和压入操作数，因此无需显式指定地址.且堆栈机也只有ADD指令

    10.2.2 An Example¶

    Suppose we want to evaluate (A + B) ⋅(C + D), where A contains 25, B contains 17, C contains 3, and D contains 2, and store the result in E.

    LD      R0,A
LD      R1,B
ADD     R0,R0,R1
LC      R2,C
LD      R3,D
ADD     R2,R2,R3
MUL     R0,R0,R2
ST      R0,E

    10.2.3 OpADD，OpMult，OpNeg¶

    1. OpAdd, which will pop two values from the stack, add them, and push the result onto the stack.

    OpAdd，将从堆栈中弹出两个值，将它们相加，然后将结果压入堆栈。

    1. OpMult, which will pop two values from the stack, multiply them, and push the result onto the stack.

    OpMult，将从堆栈中弹出两个值，将它们相乘，然后将结果推入（push）堆栈。

    1. OpNeg, which will pop the top value, form its 2’s complement negative value, and push the result onto the stack. This will allow us to subtract two numbers A minus B by first forming −B and then adding the result to A

    OpNeg，它将弹出顶部值，形成其 2 的补负值，并将结果压入堆栈。这将允许我们通过首先形成 −B 然后将结果添加到 A 来减去两个数字 A 减去 B

    10.2.3.1 The OpAdd Subroutine¶

    前提条件：If the result is within the range of acceptable values (i.e., an integer between −999 and +999), then the result is pushed onto the stack.

    计算无法成功情况：

    • Fewer than two values are available on the stack for source operands.堆栈上可用于源操作数的值少于两个
    • the result is out of range.结果超出范围

    在这两种情况下，堆栈都会恢复到 OpAdd 子例程开始时的状态。如果第一次弹出不成功，则堆栈不会更改，因为 POP 例程使堆栈保持原样。如果两次弹出中的第二次报告失败，则堆栈指针递减，这实际上返回弹出到堆栈顶部的第一个值。如果结果超出可接受值的范围，则堆栈指针递减两次，将两个值返回到堆栈顶部。

    ;Subroutine to pop the top two elements from the stack,add them and push the sum onto the stack.
;R6 is the stack pointer

OpAdd       ST      R0,SavedR0
            ST      R1,SavedR1
            ST      R5,SavedR5
            ST      R7,SavedR7
            JSR     POP             ;跳转到POP，获得第一个源操作数
            ADD     R5,R5,#0        ;Tset POP是否成功
            BRp     OpAdd_Exit      ;如果pop失败，就跳转到该分支
            ADD     R1,R0,#0        ;为第二个操作数提供空间
            JSR     POP             ;获得第二个源操作数
            ADD     R5,R5,#0        ;TEST
            BRp     OpAdd_Restore1  ;不成功，恢复第一个数
            ADD     R0,R0,R1        ;ADD相加操作
            JSR     RangeCheck      ;检查result的size，是否越界
            ADD     R5,R5,#0        ;都以R5的值作为检测的指标
            BRp     OpAdd_Restore2  ;out of range ，恢复
            JSR     PUSH            ;PUSH the sum on the stack
            BRnzp   OpAdd_Exit      ;无条件跳转结束
OpAdd_Restore2
            ADD     R6,R6,#-1
OpAdd_Restore1
            ADD     R6,R6,#-1       ;指针回退
OpAdd_Exit  
            LD      R0,SavedR0
            LD      R1,SavedR1
            LD      R5,SavedR5
            LD      R7,SavedR7
SavedR1     .BLKW   #1
SavedR0     .BLKW   #1
SavedR5     .BLKW   #1
SavedR7     .BLKW   #1

;发现目前为止，还缺少POP，PUSH和RangeCheck


    ;Subroutine to check that a value is between -999 and +999
RangeCheck
        LD      R5,NEG999
        ADD     R5,R0,R5        ;R0 contains the result
        BRp     BADRANGE
        LD      R5,POS999
        ADD     R5,R0,R5
        BRn     BADRANGE
        AND     R5,R5,#0        ;表示result在范围内，success

    10.2.3.2 The OpMult Subrountine¶

    由于LC-3没有multiplication，所以乘法是通过执行一系列的加法实现的。

    如果两次弹出中的第二次报告失败，则堆栈指针递减，这实际上返回弹出到堆栈顶部的第一个值。如果结果超出可接受值的范围（如之前一样，R5 中将用 1 表示），则堆栈指针递减两次，将两个值返回到堆栈顶部。

    ;Two values are popped from the stack,multiplied,and if their product is within the acceptable range,the result is pushed onto the stack.
;R6 is the stack pointer

OpMult
        ST      R0,SavedR0
        ST      R1,SavedR1
        ST      R2,SavedR2
        ST      R3,SavedR3
        ST      R5,SavedR5
        ST      R7,SavedR7
        AND     R3,R3,#0        ;R3 holds sign of multiplier R3保存乘数的符号
        JSR     POP             ;R0,得到第一个乘数
        ADD     R5,R5,#0
        BRp     OPMULT_EXIT     ;如果pop失败，就结束程序
        ADD     R1,R0,#0        ;make room for next POP.因为第二个数也是保存在R0，会覆盖第一个数。R1保存第一个乘数的值
        JSR     POP             ;获得第二个源操作数
        ADD     R5,R5,#0
        BRp     OPMULT_RESTORE1     ;如果pop失败，就结束程序
        ADD     R2,R0,#0        ;移动乘数，判断正负号。R2保存第二个乘数的值
        BRzp    POSMULTIPLIER   ;正数
        ADD     R3,R3,#1        ;建立一个flag，假设是负数
        NOT     R2,R2           ;如果R2是负数，即某一个乘数是负数，则令R3 = 1，后续对乘数取反操作
        ADD     R2,R2,#1        ;R2 = -multiplier。补码形式，得到相反数，逐位取反再加一
POSMULTPIER
        AND     R0,R0,#0        ;如果是正数，则不做取相反数操作
        ADD     R2,R2,#0        ;R1保存第一个乘数，R2保存第二个乘数，且目前是正数
        BRz     PUSHMULT        ;如果第二个乘数是0，则直接输出0
MULTLOOP    
        ADD     R0,R0,R1        ;R0初始状态是0，保存和输出结果用
        ADD     R2,R2,#-1       ;
        BRp     MULTLOOP
        ADD     R3,R3,#0
        BRz     PUSHMULT        ;如果R3 = 1，说明原本是负数，需要进行取相反操作。如果R3 = 0，则说明第二个乘数本身就是正数
        NOT     R0,R0
        ADD     R0,R0,#1
PUSHMULT
        JSR     PUSH            ;输出结果用
        BRnzp   OPMULT_EXIT
OPMULT_RESTORE1
        ADD     R6,R6,#-1
        ADD     R6,R6,#-1
OPMULT_EXIT 
        LD      R0,SavedR0
        ......
        RET
SavedR0 .BLKW   #1
......

    10.2.3.3 The OpNeg Subroutine¶

    要与堆栈顶部的两个元素执行减法，我们首先将堆栈顶部的元素替换为其负数，然后使用 OpADD。也就是说，如果栈顶包含A，而栈上第二个元素包含B，我们可以通过先对栈顶取反，然后执行OpAdd，将B−A压入栈。

    ;Subroutine to pop the top of the stack,form its negative.
;push the result onto the stack
OPNEG
        ST      R0,SavedR0
        ST      R5,SavedR5
        ST      R7,SavedR7
        JSR     POP
        ADD     R5,R5,#0
        BRp     OPNEG_EXIT
        NOT     R0,R0
        ADD     R0,R0,#1
        JSR     PUSH
OPNEG_EXIT
        LD      R0,SavedR0
        LD      R5,SavedR5
        LD      R7,SavedR7
        RET
SaveR0  .BLKW   #1

    10.3 The Calculator¶

    我们将使用键盘告诉计算器要做什么。我们可以输入最多三位小数的正整数、算术运算符 +（用于 ADD）、*（用于 MUL）和 −（用于负数），以及三个附加命令 D（在监视器上显示计算结果）、C（删除所有输入的值）和 X（关闭计算器）。

    • X Exit the simulation.
    • D Display the value at the top of the stack.

    • C Clear all values from the stack.

    • +Pop the top two elements A,B off the stack and push A+B.

    • ** Pop the top two elements A,B off the stack and push A* * B.

    • − Pop the top element A off the stack and push “minus” A
    • Enter 或 LF 将键盘上键入的值推入堆栈顶部。如果用户想要输入数字，则输入数字（最多三位小数），然后按 或 <换行 (LF)>。

    eg: (51 - 49) * (172 + 205) - (17 * 2)

    5,1,LF,4,9,LF,−,+,1,7,2,LF,2,0,5,LF,+, * ,1,7,LF,2,LF,*,−,+,D.

    计算器算法的工作原理如下：我们使用键盘输入命令和小数值。我们使用监视器来显示结果。我们使用堆栈来保存用于执行算术运算的源操作数以及这些算术运算的结果，如第 10.2 节中所述。输入和显示的值仅限于三位小数，即仅限 -999 和 +999（含）之间的值
