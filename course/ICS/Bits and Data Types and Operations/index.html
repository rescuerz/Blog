    Chapter2 | Bits and Data Types and Operations¶

    2.1 Bits and Data Types¶

    The bit as the unit of information

    • 0 或 1
    • The bit is the unit of information. With one wire, one can differentiate only two things.

    Data Type

    • unsigned integer
    • signed integer
    • logical variable
    • float
    • ASCII

    2.1.1 unsigned integer and signed integer（原码）¶

    对于unsigned integer

    • k bits,取值范围\([0,2^{k}-1]\)

    对于signed integer

    • 第一位是符号位，1 表示 负数，0 表示 正数
    • k bits ，
    • 最大数\(01111\cdots1 = 2^{k-1}-1\)
    • 最小数\(1111\cdots1=-2^{k-1}+1\)
    • 所以取值范围\([-2^{k-1}+1,2^{k-1}-1]\),存在相同的数\(100\cdots0,000\cdots0\)

    2.1.2 一的补码1’s Complement（反码）¶

    • 对于一个非负数，它的反码是本身

    • 对于一个负数，它的反码是，它的相反数的二进制串按位取反，或者signed位不动，其他位按位取反 $$ eg:\ 5 = 0101,5的补码\ -5 = 1010,-5的补码 $$

    2.1.3 二的补码 2‘s Complement¶

    \[ n位，a_{n-1}对应的的权重是-2^{n-1},其余a_{n-2},a_{n-3},\cdots,a_0,对应的权重分别为正数\\ \]
    • 因此，n位补码表示的范围是\([-2^{n-1},2^{n-1}-1]\)
    \[ 5 = 0101 \\ -5 = 1011 \]
    • 一个数相反数的补码是按位取反再加1
    • 会发现两个相反数相加会得到10000，即会多一位，其余全是0，得到的进位可以忽略
    \[ 0 = 00000\\ 1 = 00001\\ -1 = (11110)_2+1 = 11111\\ -1 + 1 = 11111 + 00001 = 100000 \]

    假设n = 4,-8的补码是1000，8的补码不存在

    2.1.4 FLoating Point Numbers¶

    浮点数由三个部分组成

    sign exponent fraction
    float 1 8 23
    double 1 13 52

    这样的位码得到的数是\(N = (-1)^S \times M \times 2^E\)

    \(N = (−1)^S × 1.fraction × 2^{exponent−127}, 1 ≤exponent ≤254\)

    取值范围是\([-32768=-2^{15},32767 = 2^{15}-1]\)

    • S : sign bit.0 表示正数，1表示负数
    • E ：阶码。\(E = exp - Bias\) , exp 就是exponent部分表示的无符号数，Bias 是一个固定的数，对于float而言是127，对于double而言是1023
    • M：尾数。M = 1.frac，frac就是fraction部分的二进制串
    \[ eg:如果表示用浮点数表示0.625\\ a. \ 0.625用二进制可以表示为(0.101)_2\\ b. \ 确定符号位sign = 0，为正数 \\ c. \ 由于尾数的形式M=1.frac，所以表示成相似形式(1.01)\times2_{-1}\\ d. \ 由此可知M = 1.01，E=-1，所以exp = 126,\\ e. \ exp 对应的无符号数为01111110，\\ f. \ fraction对应的二进制串为01000\cdots00,合计23位\\ 最终结果为0 \ 01111110 \ 01000\cdots0 \]
     What does the floating point data type 00111101100000000000000000000000 represent?

1. The leading bit is a 0. This signifies a positive number. 
2. The next eight bits(01111011) represent the unsigned number 123. If we subtract 127, we get the actual exponent −4. 
3. The last 23 bits are all 0. Therefore, the number being represented is+1.000000000000000000000000 ⋅2^(−4), which is  1/16

    特殊情况：

    1. exp = 0时，规定M = 0.frac，E = 1-bias = -126，其中frac = 0时，表示的数字是0.0

    N = （-1）^S * 0.frac * 2^(-126)

    1. exp = 11111111时，

    若frac = 0，则这个数表示 正无穷 +inf或者负无穷 -inf

    若frac 不等于0，则这个数表示NaN（Not a Number）

    e.g. 1 11111111 0000000000000000001

    ​ 0 00000000 1100000000000000000

    第一个数阶码exp全为1，尾数frac不为0，因此为NaN，not a number

    第二个数阶码exp全为0，M=0.frac =（0.11）_2，答案为0.75*2^(-126)

    2.2 Arithmetic¶

    • 对于加减法

    • 逢二进一

    • 两个二进制数位数不同的时候需要先将位数少的那个数符号位进行扩展成相同的位数

    • signed-extending符号扩展，对于正数而言，在高位不断填充0直至位数相同，对于负数而言，在高位不断填充1

      • 对于正数而言，前面添加无穷多个0，都不影响结果，补码情况下，一个数的相反数表示方法为按位取反再加1.所以前面添加的0全部转化为1

      • 再来思考一下为什么负数前面添加1，不会影响结果。eg：-5 = 1111011或者1011（-8+2+1），在前面添加1之后，第一位为负数，此处为-64，第二位为32，两者相加得到-32，可以看作是第一位的1不存在，以此类推，直到1之后出现0，便可得到相反数的二进制表示

    • 计算\(A-B\)，相当于计算\(A+(-B)\),即先计算B的相反数，（在补码情况下，只需按位取反再加1），再将B的相反数和A相加

    $$ \begin{aligned} & 11的二进制表示为01011\ & 3 的二进制表示为00011\ & 11+3=14的二进制表示为01110\ &对于减法\ & 11-3 = 11+(-3)\ & -3的二进制表示为按位取反加一，即11101\ & 11-3 = 8的二进制表示为：\ & 01011\ & 11101\ &101000\ &忽略进位，即可得到最终结果为01000 \end{aligned} $$

    • overflow溢出：当两个正数相加得到的结果为负数，或者两个负数相加的结果是正数

      • eg：假设n=4，计算\(0111(7)+0010(2) = 1001(-7),补码情况\)
      • 一般情况下，n位bits，补码的取值范围是\([-2^{n-1},2^{n-1}-1]\),溢出即超过了范围

    • 十进制与二进制的转化

    • 注意一下，小数如果用二进制表示即可

    • eg: \((0.101)_2 = 0.625\)

      小数的第一位是0.5（\(2^{-1}\)）,后续依次为0.25（\(2^{-2}\)），0.125(\(2^{-3}\))
