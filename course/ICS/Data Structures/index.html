    Chapter 8 | Data Structures 数据结构¶

    In this chapter, we will study three abstract data types: stacks, queues, and character strings(栈，队列，字符串)

    8.1 Subroutines函数¶

    For all of these reasons, it is good to have a way to use program fragments efficiently. Such program fragments are called subroutines, or alternatively, procedures, or in C terminology, functions.

    出于所有这些原因，最好有一种方法来有效地使用程序片段。此类程序片段称为子例程，或者称为过程，或用 C 术语称为函数。

    8.1.1 The Call/Return Mechanism 调用/返回机制¶

    With subroutine，programmer has to provide the code A only once. 对于相同的代码片段，只需提供一次，其他的就只要call/return

    The programmer uses the call/return mechanism to direct the computer each time via the call instruction to the code A, and after the computer has executed the code A, to the return instruction to the proper next instruction to be executed in the program.

    程序员使用调用/返回机制，每次通过调用指令将计算机引导到代码A，并在计算机执行完代码A后，将返回指令引导到程序中正确的下一条要执行的指令。

    • The program that contains the call as the caller
    • The subroutine that contains the return as the callee

    The call/return mechanism consists of two instructions.

    调用/返回机制由两条指令组成

    1. The first instruction JSR(R) is in the caller program anddoes two things:

    2. It loads the PC with the starting address of the subroutine

      将函数的起始地址加载到 PC 中

    3. and it loads R7 with the address immediately after the address of the JSR(R) instruction.（将紧随 JSR(R) 指令地址之后的地址加载到 R7 中）

    4. The address immediately after the address of the JSR(R) instruction is the address to come back to after executing the subroutine

      紧接在 JSR(R) 指令地址之后的地址是执行子程序后要返回的地址。. We call the address we come back to the return linkage（返回链接）.

    5. The second instruction JMP`` R7 is the last instruction in the subroutine (i.e., in the callee program). It loads the PC with the contents of R7, the address just after the address of the JSR instruction, completing the round trip flow of control from the caller to the callee and back.

    6. R7中存放着的是JSR指令地址后的地址

    7. 实现在函数执行完成之后，按照顺序跳转到下一条指令（安排）

    JSR A 跳转到子程序的开始

    结束子程序后，我们可以用RET或者JMP R7返回原来要执行的指令，RET指令完全等价于JMP R7

    8.1.2 JSR（R）——The Instruction That Calls the Subroutine¶

    关于跳转指令JAR（R），根据寻址的方式PC-relative 和 Base Register addressing，分为JSR和JSRR，用于跳转到函数

    JSR(R)的功能：

    1. 计算子程序函数的起始地址，并加载到PC中
    2. 将返回地址保存在R7中，返回地址是JSR（R）后面的指令的地址

    • Bits[15:12] 表示opcode 0100
    • Bit[11] 表示寻址方式，1表示寻址方式PC-relative，o表示寻址方式为Base Register addressing

    JSR

    The JSR instruction computes the target address of the subroutine by signextending the 11-bit offset (bits [10:0]) of the instruction to 16 bits and adding that to the incremented PC.

    JSR 指令通过将指令的 11 位偏移量（位 [10:0]）符号扩展为 16 位并将其添加到递增的 PC 来计算子例程的目标地址。

    eg

    JSR的地址是x4200，PC + offset = x4201 + xFC04 = x3E05

    返回x4201地址保存在寄存器R7中

    JSRR

    已知JSRR指令的地址是x420A，R5是基址寄存器，contains x3002

    PC will be loaded with x3002 + x0000 = x3002

    返回x420B保存在寄存器R7中

    8.1.3 Saving and Restoring Registers保存和恢复寄存器¶

    Every time an instruction loads a value into a register，the value that was previously in that register is lost

    当指令将值value保存在寄存器时，寄存器原先保存的值就会消失

    由于子程序通过执行第 05、06 和 0A 行中的指令破坏了 R1、R2 和 R3 中的值，因此当 CALLER 在子程序第 20 行的 JMP R7 指令之后恢复执行时，这些值将丢失给 CALLER。

    为了阻止执行函数后R1，R2，R3的值发生改变

    在初始化阶段，将R1的值写入内存SaveR1中……，子例程中的三个位置（第 22、23 和 24 行）已预留用于保存这些寄存器值。

    后续在1D，1E，1F行，又将存储在SaveR1，SaveR2……的值重新lD进入寄存器中

    在子例程使用 R1、R2 和 R3 供自己使用之前，子例程会保存调用程序放入其中的值。并且，在子例程返回到调用程序之前，这些值被放回（即恢复）调用程序它们的位置。

    1. We call this techniquecallee savebecause the subroutine (i.e., the callee) saves and restores the registers

    此技术为callee save被调用者保存，因为被调用者（函数）保存和恢复寄存器

    在函数（被调用程序，called program）中进行保存和恢复

    1. caller savebecause the calling program saves and restores the register vallue

    调用者保存，因为调用程序保存并恢复寄存器值

    由于 JMP START 在 R7 中加载返回链接，因此 JMP START 指令的执行会破坏 R7 中的任何内容。因此，如果调用程序在 START 处调用子例程之前已在 R7 中存储了一个值，并且在==从子例程返回后需要该值==，则调用程序必须保存并恢复 R7。

    在函数外（调用程序，calling program），JSR发生之前，保存，发生之后，恢复

    如果需要保存寄存器中的原先值，一般情况下，先将寄存器中的值ST存储在内存中，再将其LD加载到寄存器中恢复它

    8.1.4 Library Routines（库例程，或者函数库）¶

    If the library routine starts at address SQRT, and the argument is provided in R0 to the library routine, and the result is obtained in R0 from the library routine

    库例程从地址 SQRT 开始，并且参数通过 R0 中提供给库例程，并且结果从库例程返回 R0 中获得

    The pseudo-op .EXTERNAL. We already saw in Section 7.4.2 that this pseudo-op tells the assembler that the label (SQRT), which is needed to assemble the .FILL pseudo-op in line 19, will be supplied by some other program fragment (i.e., module) and will be combined with this program

    伪操作，.EXTERNAL 这个伪操作告诉汇编器，和其他程序片段program fragment合并在一起计算

    The executable image is produced at link time.

    可执行映像是在链接时生成的。在链接时组合多个模块以生成可执行映像

    8.2 Stack¶

    the defining notion of a stack is that the last thing you stored in the stack is the first thing you remove from it.

    在堆栈中存储的最后一个内容是您从堆栈中删除的第一个内容

    Last in,First out,LIFO

    In Figure 8.8a, the stack is initially shown as empty. Access is always via the first element, which is labeled TOP.

    If the value 18 is pushedonto the stack, we have Figure 8.8b.

    If the three values 31, 5, and 12 are pushed (in that order), the result is as shown in Figure 8.8c.

    Finally, if two values are popped from the stack, we have Figure 8.8d.

    A distinguishing feature of the stack of Figure 8.8 is that, like the quarters in the coin holder, as each value is added or removed, all the other values already on the stack move.

    8.2.1 Implementation in memory¶

    如图这是最常见的堆栈实现

    This stack consists of a sequence of memory locationsalong with a mechanism（机制）, called the stack pointer(堆栈指针), which keeps track of the top of the stack. We use==R6 to contain the address of the top of the stack. That is, in the LC-3, ==R6 is the stack pointer.

    该堆栈由一系列内存位置以及称为堆栈指针的机制组成，该机制跟踪堆栈的顶部。我们使用 R6 来包含栈顶的地址。即在LC-3中，R6是堆栈指针

    在上图中，x3FFB，x3FFC，x3FFD，x3FFE，x3FFF，是五个提供给堆栈的内存。

    The actual locations comprising the stack at any single instant of time are the consecutive locations from x3FFF to the location specified in R6, that is, the top of the stack.

    在任何单个时刻组成堆栈的实际位置是从 x3FFF 到 R6 中指定的位置（即堆栈顶部）的连续位置。

    1. 在Figure 8.9（a），Since there are no values on the stack,the stack pointer contains the address x4000

    由于堆栈上没有值，因此堆栈指针包含地址 x4000

    1. 在Figure8.9（b），the stack push the value 18，the stack pointer contains the address x3FFF，which is the new top of the stack

    在堆栈pushvalue18后，堆栈指针位于x3FFF，成为堆栈的top

    意味着，push或者pop，会导致堆栈指针的位置的变化

    1. 在Figure8.9（c），the stack push the value 31，5，12，in this order。

    Note that the values inserted into the stack are stored in memory locations having decreasing addresses. We say the stack grows toward zero

    插入堆栈的值存储在地址递减的内存位置中。我们说堆栈向零增长

    1. 在Figure8.9（d），shows the stack after popping the top of two elements off the stack

    those two elements (the values 5 and 12) that were popped are still present in memory locations x3FFD and x3FFC.

    被弹出的这两个元素（值 5 和 12）仍然存在于内存位置 x3FFD 和 x3FFC 中。已经存储在堆栈上的数据不会物理移动

    后续介绍两种堆栈操作push 和 pop

    8.2.1.1 push¶

    在Figure 8.9（a），初始状态，堆栈为空，指针位于x4000。后续将18push进入堆栈，堆栈指针R6递减，此时R6保存的地址是x3FFF，对应的是要存储压入堆栈的值的位置。

    The actual push is done by first loading 18 into R0, and then executing STR R0,R6,#0. This stores the contents of R0 into memory location x3FFF.

    存储的过程是，先将18保存到R0，再执行STR R0，R6，#0，将R0的内容18写入R6保存的地址x3FFF中。

    to push a value onto the stack, we first load that value into R0. Then we decrement R6, which contained the previous top of the stack. Then we execute STR R0,R6,#0, which stores the contents of R0 into the memory location whose address is in R6.

    8.2.1.2 pop¶

    弹出堆栈顶部包含的值并将其加载到 R0 中。堆栈指针（R6）递增，表示堆栈顶部的旧值已被弹出，不再在堆栈上，并且堆栈顶部有一个新值。

    8.2.1.3 Underflow下溢¶

    堆栈上只剩下两个值，但是尝试从堆栈中弹出pop三个值

    we can test for underflow by comparing the stack pointer with x4000, which would be the contents of R6 if there were nothing left on the stack to pop

    我们可以通过将堆栈指针与 x4000 进行比较来测试下溢，如果堆栈上没有剩余内容可以弹出，这将是 R6 的内容

    如果pop不成功，希望pop routine return to the calling program with the underflow information containd in a register让 POP 例程返回到调用程序并在寄存器中包含下溢信息

    1. Upon return from the POP routine, the calling program would examine R5 to determine whether the POP completed successfully(R5 = 0), or not (R5 = 1).

    如果R5 = 0，说明pop成功，如果R5 = 1，说明pop失败

    1. 如果pop成功与否反映在R5身上，所以R5原先的值会丢失，所以，如果需要R5原先的值，就需要实现进行保存，看之前的8.13 Saving and Restoring Register

    • AND R5初始化为0
    • LD R1存储地址xC000,由x4000按位取反再加1得到
    • ADD R2 = R6 + R1,此处是为了判断R6，堆栈指针是否在x4000
    • BR 若R2 = 0，说明堆栈指针在x4000处，此时堆栈为空，跳转到Failure，R5 = R5 + 1 = 1表示pop失败，return
    • 如果 R2 不为0，则pop成功，堆栈指针R6 加1

    8.1.2.4 overflow¶

    • We augment the PUSH routine with instructions to store 0 (success) or 1 (failure) in R5, depending on whether or not the push completed successfully.

    用指令扩充push 例程routine，在R5中存储0，表示push成功，在R5中存储1，表示push失败

    • Upon return from the PUSH routine, the calling program would examine R5 to determine whether the PUSH completed successfully (R5 = 0) or not (R5 = 1).

    从push例程返回后，调用程序将检查R5来确定push是否成功完成

    要看push是否成功是==看top是否和x3FFB（最上方，最小的地址）相等==

    • AND R5初始化
    • 在R1中存入 MAX，即x3005，为x3FFB的相反数
    • ADD R2 = R6 + R1
    • BR 进行判断，如果R2 = 0，则说明此时top堆栈指针恰好位于x3FFB，所以无法进行push操作，跳转到Failure，对寄存器R5+1，表示push失败
    • ADD 如果R2不等于0，push加入一个数，则堆栈指针-1，上移一位
    • STR R0储藏着push进入的数，通过STR操作，将R0中保存的数，写入堆栈指针所在的内存
    • 最后RET return

    8.2.2 The Complete Picture¶

    • If we wish to push a value onto the stack, we simply load that value into R0 and execute JSR PUSH.

    如果我们希望将一个值压入堆栈，我们只需将该值加载到 R0 中并执行 JSR PUSH 即可。

    • To pop a value from the stack into R0, we simply execute JSR POP.

    要将一个值从堆栈弹出到 R0 中，我们只需执行 JSR POP 即可

    • If we wish to change the location or the size of the stack, we adjustBASE and MAX accordingly.

    如果我们希望更改堆栈的位置或大小，我们可以相应地调整 BASE 和 MAX。

    由于push和pop函数（subroutines）会使用R1和R2，即改变R1和R2的值，所以我们需要在使用R1和R2之前保存它们，在返回调用程序（calling program）之前恢复它们

    PUSH 和 POP 例程也写入 R5。但是，正如我们已经指出的，调用程序知道子例程将在 R5 中报告成功或失败，因此如果需要存储在 R5 中的值，则调用程序的工作是在执行 JSR 指令之前保存 R5之后。正如第 8.1.3 节中所讨论的，这是调用者保存的示例。

    ;Subroutines for carrying out the PUSH and POP functions
;program works with a stack consisting of memory locations x3FFF through x3FFB.
;R6 is the stack pointer.
POP     AND R5,R5,#0      ;初始化R5，表示成功
        ST  R1,Save1
        ST  R2,Save2      ;事先保存R1，R2的value到指定地址
        LD  R1,EMPTY      ;EMPTY contains x4000的相反数
        ADD R2,R6,R1      ;堆栈指针和x4000进行比较，保存在R2中
        BRz fail_exit     ;Branch if stack is empty
        LD  R0,R6,#0      ;将R6地址保存的数保存在R0中
        ADD R6.R6,#1      ;堆栈指针+1
        BRnzp success_exit
PUSH    AND R5,R5,#0      ;初始化R5，表示成功
        ST  R1,Save1
        ST  R2,Save2      ;事先保存R1，R2的value到指定地址
        LD  R1,FULL       ;FUll contains x3FFB的相反数
        ADD R2,R6,R1      ;堆栈指针和x3FFB进行比较，保存在R2中
        BRz fail_exit     ;Branch if stack is full
        ADD R6,R6,#-1
        STR R0,R6,#0
success_exit
        LD  R2,Save2
        LD  R1,Save1   ;恢复R1，R2的原先值
        RET
fail_exit
        LD R2,Save2 ; Restore original
        LD R1,Save1 ; register values
        ADD R5,R5,#1 ; R5 <-- failure
        RET
EMPTY   .FILL xC000
FULL    .FILL xC005
Save1   .FILL x0000
Save2   .FILL x0001

    8.3 Recursion，a powerful Technique When used Appropriately（递归）¶

    8.3.1 Factorial¶

    FACT    ST      R1,Save1        ;保存和恢复寄存器R1
        ADD     R1,R0,#-1       ;R1 = R0 - 1,对应n-1
        BRz     DONE            ;如果R0 = n = 1，则结束，最终结果就是1
        ADD     R1,R0,#0        ;令R1 = n，便于计算完（n-1）!后，将R1和R0相乘得到最终结果n!
        ADD     R0,R1,#-1       ;令R0 = n-1，进入后续的FACT运算，得到(n-1)!
B       JSR     FACT
        MUL     R0,R0,R1        ;相乘
DONE    LD      R1,Save1
        RET
Save1   .BLKW

    • 由于LC-3，没有MUL指令（相乘），所以需要另一个子例程调用

    • 此处还有一个问题，图 8.13 显示了我们希望的指令执行流程。主程序使用地址 A 处的 JSR 指令调用子例程。这将导致执行标记为 #1 的代码。在地址 B 处，子例程 FACT 使用指令 JSR FACT 调用自身。这会导致执行标记为 #2 的代码，依此类推。注意，当主程序执行 JSR FACT 指令时，返回链接 A+1 保存在 R7 中。在标记为 #1 的代码块中，地址 B (JSR FACT) 处的指令将其返回链接 B+1 存储在 R7 中，从而破坏了 A+1，因此无法返回主程序。

    • 我们可以通过在执行地址 B 处的 JSR FACT 之前将地址 A+1 压入堆栈来解决这个问题。随后返回到地址 B+1 后，我们可以弹出堆栈并将地址 A+1 加载到 R7 中，然后再执行指令 RET 返回主程序。

    • 问题加一：在#1的代码块中，指令ADD R1，R0，#0将值n加载到R1中，而在#2的代码块中，指令ADD R1，R0，#0将值n-1加载到R1中。会导致指令回流时，n被覆盖
    • 我们也可以用栈来解决这个问题。也就是说，我们不是在将 n-1 加载到 R0 之前将值 n 移至 R1，而是将 n 压入堆栈，然后在使用 (n-1) 从子例程返回后在需要时将其弹出在 R0 中。
    • 问题加二：在子例程中，第一条指令是将R1保存在Save1中，而RET之前的最后一条指令时将其恢复成R1.这样做是为了在调用子例程前后R1的值不变。但是，此处由于子例程时递归的，所以当地址B处的JSR指令重新调用子例程时，R1存储着ADD R1，R0，#0指令的结果，后续FACT的第一条指令就是保存，覆盖第一次FACT保存在Save1的值
    • 可以用栈来解决这个问题。我们只需将 ST R1,Save1 替换为推送，将 LD R1,Save1 替换为弹出。

    8.3.2 The Maze，a Good Example¶

    避免使用递归来计算阶乘或斐波那契的原因很简单，迭代算法足够简单，易于理解，而不会带来可怕的递归执行时间损失。然而，重要的是要指出，有时递归的表现力之美对于解决复杂问题很有用。以下涉及迷宫的问题就是这种情况：给定一个迷宫和迷宫内的起始位置，编写一个程序来确定是否有从起始位置离开迷宫的出路。

    迷宫的每一单元都可以通过是否有通往东西南北的门，以及是否有从单元到外界的门来表征。每一单元都可以用一个内存字表示

    • Bit[4]=1 if there is a door to the outside world; Bit[4]=0 if no door.

    bit[4]表示是否有从单元通往外界的门，有则为1

    • Bit[3]=1 if there is a door to the cell to the north; Bit[3]=0 if no door.

    bit[3]表示单元是否有北门

    • Bit[2]=1 if there is a door to the cell to the east; Bit[2]=0 if no door.

    东门

    • Bit[1]=1 if there is a door to the cell to the south; Bit[1]=0 if no door.

    南门

    • Bit[0]=1 if there is a door to the cell to the west; Bit[0]=0 if no door

    西门

    The words are stored in what we call row major order.

    That is ，row1 is stored，then row2 is stored，……

    A recursive Subroutine to Exit the Maze¶

    如果我们能退出迷宫，我们将从子程序中返回，R1=1；如果不是，我们将返回 R1=0

    该算法的工作原理如下：

    1. 在每个单元中，我们首先询问该单元是否有通往外界的出口。如果是，我们返回值1并返回。
    2. 如果没有，我们会询问是否应该尝试北边、东边、南边或西边的牢房。为了在任何方向上尝试牢房，显然在该方向上必须有一扇通往牢房的门。
    3. 此外，我们希望确保我们不会陷入无限循环，例如，有一些门允许我们向北移动一个单元格，然后从那里向东移动一个单元格，从那里向南移动一个单元格，然后从那里向西移动一个单元格，让我们回到我们开始的地方。为了防止发生这种情况，我们在访问的每个单元格中放置了一个“面包屑（breadcrumb）”，并且只有在我们之前没有访问过该单元格的情况下才会转到该单元格并使用 JSR FIND EXIT。

    a. From our cell, we ask if we can exit. If yes, we are done. We exit with R1=1.

    b. If not, we put a breadcrumb in our cell. Our breadcrumb is bit [15] of the word corresponding to our current cell. We set it to 1

    c. We ask two questions: Is there a door to the north, and have we never visited the cell to the north before? If the answer to both is yes, we set the address to the cell to the north, and JSR FIND EXIT.We set the address to the cell to the north by simply subtracting 6 from the address of the current cell. Why 6? Because the cells are stored in row major order, and the number of columns in the maze is 6

    d. If the answer to either question is no, or if going north resulted in failure, we ask: Is there a door to the east, and have we never visited that cell before? If the answer to both is yes, we set the address to the address of the cell to the east (by adding 1 to the address) and JSR FIND EXIT.

    e. If going east does not get us out, we repeat the question for south, and if that does not work, then for west.

    f. If we end up with no door to the west to a cell we have not visited, or if there is a door and we haven’t visited, but it results in failure, we are done. We cannot exit the maze from our starting position. We set R1=0 and return.

    ;Recursive subroutine that determines if there is a path from current cell to the outside world
;input:R0,current cell address
;output:R1,YES(1),NO(0)

            .ORIG   x4000
FIND_EXIT   ;save modified regsters into the stack保存修改后的寄存器至栈中
            ADD     R6,R6,#-1
            STR     R2,R6,#0        ;R6是栈的指针，R2保存调用者的单元格信息(地址对应的数据)
            ADD     R6,R6,#-1
            STR     R3,R6,#0        ;R3,holds the cell address of the caller（地址）
            ADD     R6,R6,#-1
            STR     R7,R6,#0        ;R7 holds the PC of the caller
            ;Move cell address to R3,since we need to use R0 as the input to recursive subroutine calls
            ADD     R3,R0,#0        ;R0是递归子例程调用的输入
            ;if the exit is in this cell.return YES
            LDR     R2,R0,#0        ;取出R0，对应的地址的数据
            LD      R7,EXIT_MASK    ;0000 0000 0001 0000
            AND     R7,R2,R7        ;AND 之后只保留bit[4],如果bit[4]=1,则说明单元与外界连通，输出YES
            BRnp    DONE_YES
            ;put breadcrumb in the current cell
            LD      R7,BREAKCRUMB   ;x8000 1000 0000 0000 0000
            ADD     R,R2,R7     ;首位是1，表示已经被检查过
            STR     R2,R0,#0        ;将新的单元格信息保存到输出地址的内存中

            ;check the north cell for a path to exit
CHECK_NORTH 
            LD      R7,NORTH_MASK   ;x0008 0000 0000 0000 1000
            AND     R7,R2,R7
            BRz     CHECK_EAST      ;R7 = 0,表示没有北门，后续检查东门
            LDR     R7,R3,#-6       ;R7【3】 = 1,表示有北门，则通过地址减6，将新地址的数据加载到R7，表示往北走一格
            BRn     CHECK_EAST      ;如果有一个breadcrumb in the north cell，检查东门。已经检查过没有与外界连通，
            ADD     R0,R3,#-6
            JSR     FIND_EXIT       ;递归检查北边的cell
            ADD     R1,R1,#0        ;R1是用于输出最终的结果，1表示能够离开迷宫
            BRp     DONE_YES;

CHECK_EAST  LD      R7,EAST_MASK        
            AND     R7,R2,R7        ;x0004 0000 0000 0000 0100
            BRz     CHECK_SOUTH     ;如果没有东门就检查南门
            LDR     R7,R3,#1        ;如果有东门，地址加1，因为东边的格子在右边
            BRn     CHECK_SOUTH     ;如果有breadcrumb，则检查南门，
            ADD     R0,R3,#1
            JSR     FIND_EXIT       ;R0是作为递归的输入值，递归检查东边的格子
            ADD     R1,R1,#0
            BRp     DONE_YES        ;FIND_EXIT之后会对R0做出改变，1表示成功离开迷宫

CHECK_SOUTH LD      R7,SOUTH_MASK       
            AND     R7,R2,R7        ;x0004 0000 0000 0000 0010
            BRz     CHECK_WEST      ;如果没有南门就检查南门
            LDR     R7,R3,#6        ;如果有南门，则到南边的格子去，地址加6
            BRn     CHECK_WEST      ;如果有breadcrumb，则检查南门，
            ADD     R0,R3,#6
            JSR     FIND_EXIT       ;R0是作为递归的输入值，递归检查南边的格子
            ADD     R1,R1,#0
            BRp     DONE_YES        ;FIND_EXIT之后会对R0做出改变，1表示成功离开迷宫

CHECK_WEST  LD      R7,WEST_MASK        
            AND     R7,R2,R7        ;x0004 0000 0000 0000 0100
            BRz     DONE_NO         ;没有西门，说明以上东西南北门都没有，封闭格子，出不去
            LDR     R7,R3,#-1       ;有西门，地址-1，到西边的格子
            BRn     DONE_NO         ;如果有breadcrumb，则已经检查过，没有与外界连通
            ADD     R0,R3,#-1
            JSR     FIND_EXIT       ;R0是作为递归的输入值，递归检查西边的格子
            ADD     R1,R1,#0
            BRp     DONE_YES        ;FIND_EXIT之后会对R0做出改变，1表示成功离开迷宫

DONE_NO     AND     R1,R1,#0
            BR      RESTORE
DONE_YES    AND     R1,R1,#0
            ADD     R1,R1,#1
RESTORE     ADD     R0,R3,#0        ;restore R0 from R3
            ;restore the rest of the modified registers from the stack
            LDR     R7,R6,#0
            ADD     R6,R6,#1
            LDR     R3,R6,#0
            ADD     R6,R6,#1
            LDR     R2,R6,#0
            ADD     R6,R6,#1
            RET
BREADCRUMB  .FILL   x8000           ;1000 0000 0000 0000
EXIT_MASK   .FILL   x0010           ;0000 0000 0001 0000
NORTH_MASK  .FILL   x0008           ;0000 0000 0000 1000
EAST_MASK   .FILL   x0004           ;0000 0000 0000 0100
SOUTH_MASK  .FILL   x0002           ;0000 0000 0000 0010
WEST_MASK   .FILL   x0001           ;0000 0000 0000 0001
            .END

    8.4 The Queue 队列¶

    FIFO stands for “First in First out.”

    In the context of the data structure, this means we need to keep track of two endsof the storage structure: a FRONT pointer for servicing (i.e., removing elements from the front of the queue)and a REAR pointer for entering (i.e., inserting into the rear of the queue).

    在数据结构的上下文中，这意味着我们需要跟踪存储结构的两端：用于服务的 FRONT 指针（即，从队列前面删除元素）和用于输入的 REAR 指针（即，插入排在队列的最后）

    Figure 8.25 shows a block of six sequential memory locations that have been allocated for storing elements in the queue. The queue grows from x8000 to x8005.

    图 8.25 显示了一个由六个连续内存位置组成的块，这些内存位置已分配用于存储队列中的元素。队列从 x8000 增长到 x8005。

    We arbitrarily assign the FRONT pointer to the location just before the first element of the queue. 我们任意地将 FRONT 指针分配到队列第一个元素之前的位置。

    We assign the REAR pointer to the location containing the most recent element that was added to the queue. 我们将 REAR 指针分配给包含添加到队列中的最新元素的位置。

    Let’s use R3 as our FRONT pointerand R4 as our REAR pointer.让我们使用 R3 作为 FRONT 指针，R4 作为 REAR 指针。

    Figure 8.25a shows a queue in which five values were entered into the queue. Since FRONT = x8001, the values 45 in memory location x8000 and 17 in x8001 must have been removed, and the front element of the queue is 23, the value contained in x8002.

    Front 指针在队列第一个元素之前的位置，所以存储在Front指针所在地址以及更小的地址的值，都必须被删除

    请注意，值 45 和 17 仍然包含在内存位置 x8000 和 x8001 中，即使它们已被删除。存储在存储位置中的内容不会被删除。内存位置的内容被简单地复制到目标寄存器中

    8.4.1 The Basic Operations：remove from front ,insert at rear¶

    • 要符合先进先出，从尾巴进，从头出

    Since FRONT points to the location just in front of the first element in the queue, we remove a value by first incrementing FRONT and then loading the value stored at that incremented address. 由于Front指针所在位置的value已经被remove，所以需要先进行Front指针加一

    假设要remove 23

    ADD R3,R3,#1          ;R3表示是Front指针
LDR R0,R3,#0          ;将Front指针对应的value加载到R0

    Since REAR = x8004, the last value to enter the queue is 74. The values in the queue in Figure 8.25b are 2 and 74. To insert another element (e.g., 10) at the back of the queue, the following code is executed:

    由于Rear指针所在位置的value已经insert，所以需要先将Rear指针+1，再写入对应的内存

    假设要插入10

    ADD R4,R4,#1        ;R4表示Rear指针
STR R0,R4,#0        ;将R0的值写到R4（Rear指针）对应的内存

    8.4.2 Wrap-Around¶

    当我们从队列中删除一个值时，该位置可用于存储另一个元素。我们通过允许可用的存储位置环绕来做到这一点。例如，假设我们要向队列添加 20 。由于 x8000 中没有存储任何内容（之前已删除了 45），因此我们可以在 x8000 中存储 20。

    • “Wrap-around” works by having our removal and insertion algorithms test the contents of FRONT and REAR for the value x8005.

    • If we wish toinsert, and REAR contains x8005, we know we have reached the end of our available storage and we must see if x8000 is available.

    REAR 包含 x8005，我们知道我们已经到达可用存储空间的末尾，并且我们必须查看 x8000 是否可用。(返回头部front检查是否有位置)

    • If we wish to remove, we must first see if FRONT contains the address x8005.

    如果我们想删除，我们必须首先查看FRONT是否包含地址x8005

    ;remove
LD   R2,LAST
ADD  R2,R3,R2          ;计算R3-x8005,
BRnp SKIP_1            ;只要R2不为0，即Front指针不在x8005，即此时队列不为空，就跳转到SKIP_1,先执行Front指针加1，在Front指针所在地址的数据加载到R0
LD   R3,FIRST          ;当发现Front指针已经位于x8005时，利用环绕wrap-around的性质，将Front指针移回到x8000，后续是将当前位置的数据取出，不进行Front指针加1操作
BR   SKIP_2
SKIP_1 ADD R3,R3,#1
SKIP_2 LDR R0,R3,#0
RET
LAST    .FILL x7FFB   ;LAST contains the negative of 8005,只要用于看头指针是否在x8005
FIRST   .FILL x8000   ;

    ;insert
LD   R2,LAST
ADD  R2,R4,R2            ;计算R4-x8005，判断R4是否已经到末尾
BRnp SKIP_1              ;只要R2不为0，即此时Rear指针不在x8005，队列还存在空间，就跳转到SKIP_1,先进行Rear指针+1，再将R0的数据写入到Rear指针对应的内存中
LD   R4,FIRST            ;当R2为0时，此时Rear指针在最后，就跳转到开头x8005
BR   SKIP_2              ;无条件跳转到SKIP_2,只进行数据的写入操作
SKIP_1  ADD R4,R4,#1
SKIP_2  STR R0,R4,#0
RET
LAST    .FILL x7FFB     ;LAST contains the negative of 8005
FIRST   .FILL x8000

    8.4.3 How Many Element Can We Store in a Queue?¶

    在图（d） There are four values in the queue: 2, 74, 10, and 20.这是因为，Rear指针已经到达过x8005，后续回到了x8000，并且输入保存了20

    • Suppose we insert 30 and 40 at the rear of the queue, producing Figure 8.25e. Both R3 and R4 contain the same address (x8002), and the queue is full.
    • Now suppose we start removing elements from the front of the queue. If we remove 2, which is at the front of the queue, R3 will contain the address x8003. If we remove the remaining five elements in the queue, we will have what is shown in Figure 8.25f. Note that the FRONT and REAR pointers for e and f are identi- cal,
    • Figure 8.25e describes a full queue and Figure 8.25f describes an empty queue!

    此时，当Front指针和Rear指针所在地址相同时，队列有两种可能，既可能是满的，也可能是空的。为了防止该不确定性的发生，对于分配了n个元素的空间，当队列中有n - 1元素时，我们便声明队列已满

    假设我们插入30，就会得到8.26的（a），此时队列中已经有五个元素，说明此时队列已经满了

    接着开始remove元素，直到队列为空，如图8.26（b）所示，此时Front = Rear

    当队列full时，Front = Rear + 1；当队列为empty时，Front = Rear

    8.4.4 Teat for Underflow，Overflow¶

    • If the queue is empty and we try to remove an element, we have an underflow condition.
    • If the queue is full and we try to insert an element, we have an overflow condition.

    如果队列为空并且我们尝试删除一个元素，则会出现下溢情况。如果队列已满并且我们尝试插入元素，则会出现溢出情况。

    我们首先检查队列是否为空，即 R3 = R4 是否。如果是这样，我们分支到 UNDERFLOW，在这里我们将 R5 设置为故障，恢复 R1，然后返回。如果没有，则执行代码以删除队列的前面。溢出测试类似。要将元素插入队列的末尾，我们首先递增 REAR 指针。如果这导致 FRONT = REAR，则队列已包含 n-1 个元素，这意味着队列已满，因此我们无法插入更多元素。我们递减 REAR 指针，将 R5 设置为 1，然后返回。

    R5 = 0 表示成功，R5 = 1 表示失败

    关于插入，我们首先必须确保队列未满。为此，我们递增 REAR 指针 (R4)，然后测试 REAR=FRONT。如果 REAR 指针最初是 x8005，我们通过将其设置为 x8000 来增加 REAR；也就是说，我们需要环绕。如果队列已满，我们需要将 REAR 设置回其原始值，然后返回，报告失败（R5 = 1）。如果队列未满，我们将希望插入的项目（位于 R0 中）存储在 REAR 中，然后返回，报告成功 (R5 = 0)。

    关于删除，我们首先通过测试是否 REAR=FRONT 来确保队列不为空(无需指针加1操作)。如果 REAR=FRONT，则队列为空，因此我们返回并报告失败。如果 REAR 与 FRONT 不同，则队列不为空，因此我们可以删除前面的元素。为此，我们首先测试 FRONT=x8005 是否。如果是，我们设置 FRONT=x8000。如果不是，我们增加 FRONT。在这两种情况下，我们都会将该内存位置中的值加载到 R0 中，然后返回，报告成功。

    ;Input:R0 for item to be inserted.R3 is Front,R4 is Rear
;Output:R0 for item to be removed
INSERT  ST      R1,SaveR1   ;操作之前事先进行保存
        AND     R5,R5,#0    ;R5的初始化
        LD      R1,NEG_LAST ;R1保存-x8005，目的是检查Rear指针是否位于x8005
        ADD     R1,R1,R4    ;
        BRnp    SKIP1       ;跳过wrap around
        LD      R4,FIRST    ;返回起点x8000
        BR      SKIP2       ;不需要进行Rear指针+1
SKIP1   ADD     R4,R4,#1    ;Rear指针+1
SKIP2   NOT     R1,R4
        ADD     R1,R1,#1    ;R1此时表示-Rear
        ADD     R1,R1,R3    ;R1 = Front - Rear,在Rear指针+1之后，判断Rear和Front是否相等，相等则队列已满
        BRz     FULL        ;若队列已满，则Rear指针回退，分Rear指针是否在x8000进行考虑
        STR     R0,R4,#0    ;不相等情况，在Rear指针对应的内存写入R0
        BR      DONE        ;跳转到结束
FULL    LD      R1,NEG_FIRST;R1 = -x8000,判断R4是否在x8000位置
        ADD     R1,R1,R4    ;R1 = Rear - x8000
        BRnp    SKIP3       ;如果不在，就进行简单的回退操作，并且在R5中写入1，表示失败
        LD      R4,LAST     ;如果Rear指针在x8000，说明原先Rear指针在x8005，退回原样
        BR      SKIP4
SKIP3   ADD     R4,R4,#-1
SKIP4   ADD     R5,R5,#1
        BR      DONE
REMOVE  ST      R1,SaveR1   ;操作之前，事先保存R1,在结束时，恢复
        AND     R5,R5,#0
        NOT     R1,R4
        ADD     R1,R4,#1    ;得到R1 = -Rear
        ADD     R1,R1,R3    ;R1 = Front - Rear，用于判断队列是否为空
        BRz     EMPTY       ;若Front = Rear，则说明，队列为空，跳转到EMPTY操作
        LD      R1,NEG_LAST ;用于判断Front指针是否为x8005
        ADD     R1,R1,R3    ;R1 = Front - x8005
        BRnp    SKIP5       ;若Front指针不在x8005，只需Front指针加一，再把当前Front指针所在地址的数据加载到R0
        LD      R3,FIRST    ;若FROnt指针在x8005，首先得使Front指针位于x8000，再把x8000的数据加载到R0
        BR      SKIP6
SKIP5   ADD     R3,R3,#1    ;Front指针+1
