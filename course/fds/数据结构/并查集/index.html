    等价关系：对称，自反，传递

    等价类：集合划分

    操作：

    1. 查：找当前集合属于哪一个等价类
    2. 并：合并两个等价类

    实现：树

    • Union, Find操作仅涉及fa，故只需存储fa
    • 静态链表建树，S[element] = fa, s[root] = 0，set name = root index

    • SetUnion

    问题：合并的最坏情况导致其退化为单链，find操作\(T(N)=O(N)\).

    解决1：按大小合并

    实现：

    • 集合大小记录：S[root] = -size;
    • 由于节点编号均为正数，故使用负数来记录大小信息

    引理：按大小合并时\(\text{height}(T)\leqslant\lfloor\log_2N\rfloor+1\)

    时间复杂度：\(N\)次合并，\(M\)次查找，\(O(N+M\log_2N)\)

    解决2：按高度合并

    实现：S[root] = -height

    路径压缩：让所有儿子都直接指向根，find操作复杂度降为\(O(1)\)

    实现：在find同时压缩

    SetType  Find ( ElementType  X, DisjSet  S ) {
    if ( S[ X ] <= 0 ) return  X;
    else return  S[ X ] = Find( S[ X ], S );
}

    循环版本：

    SetType  Find ( ElementType  X, DisjSet  S )
{   ElementType  root,  trail,  lead;
    for ( root = X; S[ root ] > 0; root = S[ root ] )
        ;  /* find the root */
    for ( trail = X; trail != root; trail = lead ) { 
    // lead带着trail将路径上所有节点连上root
       lead = S[ trail ] ;   
       S[ trail ] = root ;   
    }  /* collapsing */
    return  root ;
}

    Union-by-height无法和路径压缩合并使用：路径压缩导致树高改变

    按秩（集合大小等指标）合并 + 路径压缩：

    • 引理：\(k_1M\alpha(M,N)\leqslant T(M,N)\leqslant k_2M\alpha(M,N)\)
    • 其中\(\alpha\)为Ackermann函数的反函数\(\alpha\leqslant4\)
    \[ A(i,j)=\left\{\begin{array}{ll}2^j&i=1,j\geqslant1\\ A(i-1,2)&i\geqslant2,j=1\\ A(i-1,A(i,j-1))&i\geqslant2,j\geqslant2\end{array}\right. \]

    七.Union and Find 并查集¶

    并查集的基本概念：

    1. 并查集是一种树形的数据结构，用于处理一些不相交的合并及查询问题（所谓的并和查）
    2. 并查集主要由一个整型数组root[ ], 两个函数find（）和union（）构成 * root[ ]用于知道每一个点的父亲，前驱节点 * find（x）用于查找指定结点x属于哪个集合 * union（x，y）用于合并结点

    7.1 Relation¶

    Relation is symmetric, If (a, b) ∈ R, then (b, a) ∈ R.

    Relation is transitive, If (a, b) ∈ R & (b, c) ∈ R, then (a, c) ∈ R.

    Relflexive Relation is relflexive, If (a, a) ∈ R**. **

    Consistent Relation is symmetric and relflexive.

    7.2.2 Union(i, j)¶

    求并运算，将含有 a 和 b 的两个等价类合并为一个等价类

    想法：make S_i a subtree of S_j,. That is ,we can set the parent pointer of one of the roots to the other root.

    将一个根结点连接到另一棵树的根结点上，往往后者连接到前者。

    Array Data Stucture Implementation

    • S[element] = the parent of element
    • S[root] = - 1
    • set name = root index

    void SetUnion(DisjSet S, SetType Rt1, SetType Rt2)
{
    S[Rt2] = Rt1;
    // 相当于一个集合的根结点，连接到另一个集合的根结点上去
}
// 具体样例
void Union(int S[], int x1, int x2)
{
    S[x2] = x1;
}

    // Union-by-Size,Always change the smaller tree

// S [ Root ] = – size;  /* initialized to be –1 */
// 按照集合的大小进行Union
void Union_by_size(int x1, int x2， int S[])
{
    int root1 = Find(x1);
    int root2 = FInd(x2);
    // 首先得判断是否需要Union
    if(root1 == root2)
    {
        return;
    }
    else
    {
        //root1 is bigger than root2,so we Union root2 to root1
        if(S[root1] <= S[root2])
        {
            S[root2] = root1; //将第二个集合连接到第一个集合
            S[root1] += S[root2]; // S[root]表示的是结合的大小
        }
        else // root2 is bigger than root1
        {
            S[root1] = root2;
            S[root2] += S[root1];
        }
    }
}
void Union_by_height(int x1, int x2, int S[])
{
    int root1 = Find(x1);
    int root2 = FInd(x2);
    // 首先得判断是否需要Union
    if(root1 == root2)
    {
        return;
    }
    else
    {
        // 集合1的高度更大，集合二接到1身上
        if(rank[root1] > rank[root2])
        {
            pre[root2] = root1;
        }
        else if(rank[root1] < rank[root2])
        {
            pre[root1] = root2;
            // 在高度不相等的情况下，短的集合接到长的集合，集合的长度仍然是原先长的集合，短的集合height+1 <= 长的集合height
        }
        // 如果两者高度相等，则默认集合二接到集合1shen
        else
        {
            pre[root2] = root1;
            rank[root1]++;
            //只有两个高度相等时，heigtht才会增加
        }
    }
}

    In Union/Find algorithm, if Unions are done by size, the depth of any node must be no more than N/2, but not *O(logN).*

    每做一次归并，都会使小的集合深度+1，但是总的深度还是看大的集合。所以只有深度相同的归并才能使总的深度+1.

    因此深度最大为log_2{N}+1，原因在于，将集合均分为两等份，每一份的深度就是log_2{N},归并后+1.

    Time complexity of N Union and M Find operations is now O( N + M log_2 N ).****

    Union 时间复杂度是O（1），把一个集合的root变一下就好了

    Find，涉及到找root和路径压缩，所以需要logN

    Let T be a tree created by union-by-size with N nodes, then height(T)<=(log2N)+1;****

    7.2.3 Find()¶

    返回给定元素的所在的集合（等价类）

    //Recursive Version
int Find(int x)
{
    // 先判断自己是不是root
    if(root[x] < 0)
    {
        return x;
    }
    else // 若不是则开始寻找并路径压缩
    {
        return root[x] = Find(root[x]);
        // 相当于先计算Find(root[x]),再进行赋值操作
    }
}
// Iterative Version
int Find(int x, int S[])
{
    int root, trail, lead;
    for(root = x; S[root] >= 0; root = S[root]);
    // 先找到集合S的root
    for(trail = x; trail != root; trail = lead)
    {
        lead = S[trail];
       S[trail] = root;//相当于多层结构，将每一层的父节点都改为最终的根结点root
    }// 再找一遍，找的时候将每一层跟的最终跟都改掉
    return root;
}

    路径压缩的效果是，从 X 到根的路径上每一个节点都使它的父节点变成根。

    路径压缩与按大小求并完全兼容，可以同时实现，但不能与按高度求并（有时称为秩）

// union_by_height
int pre[10010]; //找父亲
int rank[10010]; // 每一个结点所在集合的height
int Find(int x)
{
    if(pre[x] == x){
        return x;
    }
    else{
        return pre[x] = Find(pre[x]);
    }
}
void Union_by_height(int x, int y)
{
    int root1, root2;
    root1 = Find(x);
    root2 = Find(y);
    if(root1 == root2)
    {
        return;
    }
    else
    {
        // 此处比较的是height，且height为正数
        if(rank[root1] > rank[root2])
        {
            pre[root2] = root1;
        }
        else if(rank[root1] < rank[root2])
        {
            pre[root1] = root2;
        }
        else
        {
            pre[root2] = root1;
            rank[root1]++;
        }
    }
}
