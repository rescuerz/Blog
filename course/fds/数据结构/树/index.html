    五.树¶

    5.1 树的定义和性质¶

    1. 树的定义： A tree is a colection of nodes.
    • (1) a distinguished node r, called the root;
    • (2) zero or more nonempty (sub)tree T1,…Tk.

    注意：

    1. Subtrees must not connect together. Therefore every node in the tree is the root of some subtree.

    2. There are N -1 edges in a tree with N nodes.

    3. Normally the root is drawn at the top.

    1. 其他定义

    • degree of a node : number of subtrees of the node. For example degree(A) = 3, degree(F) = 0.
    • degree of a tree: \(= max_{node \in tree}{degree(node)}\).For example, degree of this tree = 3
    • parent: a node that has subtrees
    • children: the roots of the subtrees of a parents
    • siblings: children of the same parent
    • leaf : a node with degree 0
    • path from n1 to nk: a unique(唯一的)sequence of nodes \(n_1, n_2, \cdots n_k\),such that \(n_i\) is the parent of \(n_{i+1}\)
    • length of path: number of edges on the path
    • depth of n_i:length of the unique path from the root to \(n_i\) Depth(root) = 0.
    • height of n_i:length of the longest path from \(n_i\) to a leaf. Height(leaf) = 0, and height(D) = 2.

    深度depth 从 root 开始计算， 高度height 从 leaf 开始计算

    • height(depth) of a tree:height(root) = depth(deepest leaf).
    • ancestors of a node:all the nodes along the path from the node up to the root
    • descendants of a node: all the nodes in its subtrees.

    二叉树是一棵树，其中每个节点的儿子不能多于两个

    5.4 线索二叉树¶

    规则：

    • 如果 Tree->Left 为空，用一个指向中序遍历中当前节点的前驱的指针代替它
    • 如果 Tree->Right 为空，用一个指向中序遍历中当前节点的后继的指针代替它
    • 这里没有空闲的指针，因此一棵线索二叉树需要有一个左儿子指针指向第一个节点的头节点。

    对于一个含n个结点的线索二叉树，共有n+1个null结点

    \[ 对于n个结点的树，改变它的形状，使得leaf结点集中在最后一行。\\ 不妨设，该树完全平铺k行，也就是2^k -1 <= n < 2^{k+1} -1\\ 此时最后一行一共n-2^k+1个结点，\\ 此时最后一行产生的NULL结点一共有2*(n-2^k+1) = 2n-2^{k+1}+2\\ 最终的计算理解为，第k行原本应该有的NULL结点数 - 第k+1行的结点数 + 第k+1行产生的NULL结点数\\ 最终，2*(2^{k-1})+n-2^k+1 = n+1 \]

    Among the following threaded binary trees (the threads are represented by dotted curves), which one is the postorder threaded tree?

    豁然开朗版：线索二叉树也有很多品种，比如inorder threaded tree，postorder threaded tree，根据不同的遍历顺序，修改定义。

    比如，此处为后序的线索树，那么定义为，如果左子树为空，就用虚线指向自己postorder前面的结点，如果前面没有，就是NULL。如果右子树为空，就用虚线指向自己postorder后面的结点，如果没有，就是NULL

    下面这个树，后序排序是：dbca 前驱就是前面的，比如b的前驱是d, d的前驱是Null 后驱就是后面的，比如d的后驱是b，a的后驱是Null d有两个指针没有用，左边指向前驱Null，右边指向b（然后就选出来B了） b的右节点有用的（已经指向d了），左节点指向其前驱，也就是d c前驱是b，后驱是a，所以左指向b，右指向a a没有指针可用了（已经左为b右为c）了 所以选B。

    5.5 complete binary tree¶

    skewed binary tree 斜二叉树

    对于complete binary tree，可以理解为，leaf集中在倒数两行，且leaf的排列需要优先满足左子树，也就是得先铺满左子树，再铺右子树。最后一行leaf的排列方式为从左往右依次排列。

    For any nonempty binary tree, $n_0 = n_2+1$where $n_0$ is thr number of leaf nodes and $n_2$ the number of nodes of degree 2.

    一个极其重要的性质：在二叉树中，叶子节点的数量等于degree为2的结点数量（有两个孩子）+1

    $$ \begin{aligned} &推导过程如下：\ &两种方式理解树的结点数量：\ &第一种从结点的类型和数量入手，首先在一棵二叉树中，结点的种类只有leaf，nodes of degree1， \ &nodes of degree2，不妨设它们的数量分别为n_0,n_1,n_2.于是总数n = n_0+n_1+n_2\ &第二种从边的数量和边是怎么产生入手，首先e = n - 1，边的数量等于结点数减一\ &leaf 产生的边的数量为0，degree 为1 的nodes产生的边数量为n_1, \ &degree 为2的nodes产生的边的数量为2n_2，于是边的总数为e = n_1 + 2n_2\ & 联立方程组： n_1 + 2n_2 = e = n-1 = n_0 + n_1 + n_2 - 1,得到n_0 = n_2 + 1 \end{aligned} $$

    5.6 Binary Search Tree¶

    定义： A binary search tree is a binary tree. It may be empty. If it is not empty, it satisfies the following properties:

    (1) Every node has a key which is an integer, and the keys are distinct.

    (2) The keys in a nonempty left subtree must be smaller than the key in the root of the subtree.

    • 若它的左子树不为空，左子树上所有节点的值都小于它的根节点。

    (3) The keys in a nonempty right subtree must be larger than the key in the root of the subtree.

    • 若它的右子树不为空，右子树上所有的节点的值都大于它的根节点。

    (4) The left and right subtrees are also binary search trees.

    右边的任意结点大于左边的结点

    这个也是二分查找树，左子树小于root同样适用于右边的树

    SearchTree MakeEmpty( SearchTree T );

    Position Find( ElementType X, SearchTree T );

    Position FindMin( SearchTree T );

    Position FindMax( SearchTree T );

    SearchTree Insert( ElementType X, SearchTree T );

    SearchTree Delete( ElementType X, SearchTree T );

    ElementType Retrieve(Position P );

    1. 查找的时间复杂度是O(depth),也就是O（logN）

    2. 最大值不一定在叶结点上，因为它的左儿子可能存在，但是右儿子不存在，依然是最大值

    3. 但是最小值一定在叶结点上，因为完全二叉树，左儿子不存在，则该节点为最小值，左儿子存在，则左儿子是最小值

    4. 看前提条件，完全二叉树，左子树的结点数量一定比右子树的结点数量多，所以中位数一定在根结点（左右字树结点数相同）或者左子树

    3.在一棵二叉搜索树上查找63，序列39、101、25、80、70、59、63是一种可能的查找时的结点值比较序列。

    F ×

    一个比较实用的判断方法是如果序列前一个元素比当前元素小，那么更新区间的最大值为当前元素；如果序列的前一个元素比当前元素大，那么更新区间的最小值为当前元素。之后的每一个元素必须在这个区间内。根据这个规则，当运行到70的时候区间已经是(70,80)，而下一个元素是59。

    \((39, +\infty),(39,101)\)此时出现25，不在区间范围内，出现错误

    Tree Insert(Tree T, int value)
{
    if(T == NULL)
    {
        T = (Tree)malloc(sizeof(TreeNode));
        T->data = value;
        T->left = NULL;
        T->right = NULL;
        return T;
    }
    else
    {
        if(value < T->data)
        {
            T->left = Insert(T->left, value);
        }
        else if(value > T->data)
        {
            T->right = Insert(T->right, value);
        }
        return T;
    }
}

{

    5.6.4 Delete¶

    Delete a leaf node : Reset its parent link to NULL.

    Delete a degree 1 node : Replace the node by its single child.

    Delete a degree 2 node :

    • Replace the node by the largest one in its left subtree or the smallest one in its right subtree.

    用左子树的最大值或者右子树的最小值进行替换

    • Delete the replacing node from the subtree.

    再将替换用的数进行删除

    Tree DeleteMin(Tree T)
{
    if(T == NULL)
    {
        return NULL;
    }
    else
    {
        if(T->left == NULL)
        {
            Tree temp = T->right;
            free(T);
            return temp;
        }
        else
        {
            T->left = DeleteMin(T->left);
            return T;
        }
    }
}
Tree DeleteMin_Iter(Tree T)
{
    if(T == NULL)
    {
        return NULL;
    }
    else
    {
        if(T->left == NULL)
        {
            Tree temp = T->right;
            free(T);
            return temp;
        }
        else
        {
            Tree parent = T;
            Tree child = T->left;
            while(child->left != NULL)
            {
                parent = child;
                child = child->left;
            }
            // now child is the min node
            // 如果child->right 存在，那么用child->right顶替child的位置
            parent->left = child->right;
            free(child);
            return T;
        }
    }
}

}

    // 删除指定值的结点
// Delete a leaf node :  Reset its parent link to NULL.
// Delete a degree 1 node :  Replace the node by its single child.
// Delete a degree 2 node :  Replace the node by the largest one in its left subtree or the smallest one in its right subtree.
// Delete the replacing node from the subtree.
Tree Delete(Tree T, int value)
{
    if(T == NULL)
    {
        printf("Element not found\n");
        return NULL;
    }
    else if(value < T->data)
    {
        T->left = Delete(T->left, value);
        return T;
    }
    else if(value > T->data)
    {
        T->right = Delete(T->right, value);
        return T;
    }
    else // value == T->data  分leaf,degree = 1, degree = 2 进行讨论
    {   
        if(T->left == NULL && T->right == NULL) // leaf
        {
            free(T);
            return NULL;
        }
        else if(T->left == NULL) // degree = 1;
        {
            Tree temp = T->right;
            free(T);
            return temp;
        }
        else if(T->right == NULL) // degree = 1;
        {
            Tree temp = T->left;
            free(T);
            return temp;
        }
        else // degree = 2;
        {
            // replace the node by the smallest one in its right subtree
            // Tree temp = Findmin1(T->right);
            // T->data = temp->data;
            // T->right = Delete(T->right, temp->data);
            // return T;
            // replace the node by the largest one in its left subtree
            Tree temp = Findmax1(T->left);
            T->data = temp->data;
            T->left = Delete(T->left, temp->data);
            return T;
        }
    }
}
